## 1. 확정된 변경 사항 및 목적 요약

'CHANGE VIEWPOINT'의 결과물을 원본과 완벽하게 100% 동일하게(Perfect Clone) 출력하기 위해, 기존의 렌더링 품질을 떨어뜨리던 보이드(Void)들을 아래 규칙으로 완벽히 차단합니다.

* **[✅ 확정] Q1 (입면도 권한 제어)**: **제안 1 채택 + 사용자 지침 적용**
  - **Phase 2 (입면도 생성)**: 원본 상세 묘사(micro-description) 텍스트와 초정밀 변수(`temperature: 0.1`)를 입면도 생성 프롬프트에도 주입하여, 부정확한 실루엣이 아닌 '정밀 도면 퀄리티'로 사전에 완성합니다.
  - **Phase 4 (최종 생성)**: "IMAGE 1(원본)은 재질/스타일의 절대 진리, IMAGE 2(입면도)는 뼈대 윤곽선 가이드로만 취급"하는 강력한 권한 분리 룰을 적용합니다.
* **[✅ 확정] Q2 (AI 창의성 통제)**: **제안 1 채택**
  - 생성 모델 API 호출 구조 상에 `generationConfig: { temperature: 0.1, topK: 1 }` 속성을 강제 하드코딩하여 뷰포인트마다 창문 배열이 뒤바뀌는 랜덤 확률 에러를 차단합니다.
* **[✅ 확정] Q3 (그림자 분리 연산)**: **제안 1 채택**
  - 프롬프트에 이미지의 짙은 그림자와 실제 페인트 텍스처를 개념적으로 분리(De-bake)하고, 새 시점 및 지정된 시간에 맞춰 태양광을 물리적으로 재계산(Shadow Casting)하라는 강력한 명령을 삽입합니다.
* **[✅ 확정] Q4 (정면 06:00 인식 도입 및 UI 미출력)**: **제안 2 채택**
  - Phase 1 인지 단계에서 AI가 각도를 내뱉기 전에 `"front_facade_evidence"` 필드를 작성하게 강제하여 180도 측면 오착각 현상을 100% 방지합니다.

## 2. 추가 질문에 대한 상세 답변 (Q4 제안 2 관련)

**💬 질문:** *"분석한 내용을 어디에 출력할지 정한 코드가 있나요?"*

**💡 답변:**
[제안 2(UI 생략)]를 채택하셨기 때문에, 앱 화면(UI) 공간 상에 해당 분석 텍스트를 출력하는 코드는 **존재하지 않게 (작성하지 않게)** 됩니다! 그 원리는 다음과 같습니다.

1. **작동 원리 (내부 코어 연산만 수행)**: `App.tsx` 내의 분석 함수(`analyzeViewpoint`) 안에서 API를 호출하면, AI가 화면 뒤편에서 `{ "front_facade_evidence": "건물 좌측면이 출입문과 발코니가 있으므로 이곳을 정면 06:00으로 설정함.", "angle": "04:30" ... }` 형태의 긴 텍스트 응답을 서버/메모리에 넘겨줍니다. 
2. **정보의 은닉 (버림 처리)**: 우리 자바스크립트 코드 단에서 `"front_facade_evidence"`라는 텍스트 데이터는 단순히 AI가 오답을 내지 않게 만들기 위해 **한 번 생각하게 만드는 용도(Chain-of-Thought)**일 뿐입니다. 화면 어디에도 이 텍스트를 할당(Bind)하지 않고 허공에 날려 버립니다. 오직 그 사유 끝에 도출된 안전한 `"angle"`과 `"elevation_parameters"` 값만 빼내어 `item.parameters`에 저장합니다.
3. 따라서 사용자 인터페이스(우측 패널 등)는 기존과 똑같이 시각적으로 깔끔하게 유지되며, 오직 백그라운드의 분석 정확도(정답률)만 비약적으로 상승하게 됩니다.

## 3. 구현 세부 계획 (코드 적용 가이드라인)

1. **Phase 1 (analyzeViewpoint) 인지 프로토콜 개편**
   - JSON 스키마 구조 최상단에 `"front_facade_evidence": "Explain exactly which side of the building is the front (06:00) based on main entrances and large windows BEFORE determining the angle."` 속성 강제 추가.
2. **Phase 2 (generateElevations) 무결성 강화**
   - API 호출 단에 `generationConfig: { temperature: 0.1, topK: 1 }` 주입.
   - 프롬프트 텍스트에 Phase 0의 극사실 묘사 `buildingDescription` 하드코딩.
3. **Phase 4 (handleGenerate) 최상단 렌더 프롬프트 개편**
   - API 호출 단에 `generationConfig: { temperature: 0.1, topK: 1 }` 주입.
   - `layerC_blindspot` 변수 등에 확정된 텍스처-그림자 분리 제어 규칙(Q1, Q3) 하드코딩.
