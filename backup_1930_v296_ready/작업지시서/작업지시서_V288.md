## 원인 분석

### Change A: SKETCH TO IMAGE 결과 배치 분기

기존(V287)에는 첫 결과냐 두 번째 이후냐에 따라 right/bottom 배치를 나누었으나, 원본과 파생을 구분하지 않았음.
업데이트된 다이어그램 및 설계 규칙에 따라, 생성 모체(소스)가 일반(최초) 아트보드인지, 파생 스케치보드인지에 따라 첫 배치 방향이 달라져야 함.

**해결**:
파생 여부를 `!currentSource.motherId` 여부 등으로 판단.
- **첫 아트보드(원본)**:
  - 형제가 없을 때: `source.x + source.width + 120`, `source.y` (우측)
  - 형제가 있을 때: `bottomMost.x`, `bottomMost.y + bottomMost.height + 120` (하단 스태킹)
- **파생 스케치**:
  - 형제가 없을 때: `source.x`, `source.y + source.height + 120` (하단)
  - 형제가 있을 때: `bottomMost.x`, `bottomMost.y + bottomMost.height + 120` (하단 스태킹)

### Change B: 두손가락 pinch 후 복귀 불가

**이벤트 순서 원인**:
첫 손가락이 선행 전환 후 `handleTouchStart` 에서 이미 변경된 'pan' 값을 이전 모드로 저장하여, 리프트 시 'pan'이 유지되는 문제가 발생.

**해결**:
`handleTouchStart`에서 `tempRestoreCanvasModeRef.current`를 우선 참조하고, 값이 있으면 해당 값을 `previousCanvasModeRef`로 이전.

### Change C: z-index 120

현재 좌측 컨트롤 바 및 우측 패널이 생성 오버레이 등에 의해 가려질 수 있음. (z-100, z-110 수준)
이를 통일되게 z-[120]으로 격상하여 모든 캔버스 오버레이 위에 명확히 표시되게 함.

### Change D: 생성 위치 stale closure 수정

`GENERATE` 시점의 참조값이 고정되어, 응답 대기 중 보드를 이동하면 과거 위치에 이미지가 생성됨.
`setCanvasItems`의 `prev` 인자에서 가장 최신 위치 요소를 찾아내어 `x`, `y` 배치를 재계산.

### Change E: `edit(+)` 아트보드 우측 배치

이미지 상에서 `edit(+)`를 호출할 때 파생 캔버스의 기본 위치 지정. 항상 소스의 우측인 `x = source.x + source.width + 120`, `y = source.y` 값을 따름.

### Change A: 배치 로직 분기 (`setCanvasItems` 내부)

```ts
if (targetMotherId) {
  const siblings = prev.filter((c: CanvasItem) => c.motherId === targetMotherId);
  const isOriginal = currentSource && currentSource.type === 'artboard' && !currentSource.motherId;

  if (siblings.length > 0) {
    // 형제가 있으면 무조건 하단 스태킹 (세로 배열)
    const bottomMost = siblings.reduce((max, cur) => cur.y > max.y ? cur : max, siblings[0]);
    newX = bottomMost.x;
    newY = bottomMost.y + bottomMost.height + 120;
  } else {
    // 형제가 없는 첫 생성일 때
    if (isOriginal) {
      // 최초 보드 -> 우측 배치
      newX = currentSource.x + currentSource.width + 120;
      newY = currentSource.y;
    } else {
      // 파생 보드 -> 하단 배치
      newX = currentSource.x;
      newY = currentSource.y + currentSource.height + 120;
    }
  }
}
```

## 동작 흐름

### 전체 프로세스 (초기 아트보드 → edit → 파생 SKETCH)

#### 1. 시작: 초기 아트보드 A 생성
```
[A] (type: artboard, motherId: null)
```

#### 2. 아트보드 A에서 첫번째 SKETCH TO IMAGE 다중 실행
- **A -> IMAGE1**: A의 우측으로 생성 (형제 없음 + 원본)
- **A -> IMAGE2**: IMAGE1의 하단으로 스태킹 (형제 있음)
- **A -> IMAGE3**: IMAGE2의 하단으로 스태킹 (형제 있음)
```
[A:SKETCH] ——→ [IMAGE1]
                |
                v
               [IMAGE2]
                |
                v
               [IMAGE3]
```

#### 3. IMAGE2에서 `edit(+)` 클릭, 파생 아트보드 B 생성
- **B**: IMAGE2의 우측으로 생성
```
               [IMAGE2] ——→ [B:SKETCH] (type: artboard, motherId: IMAGE2.id)
```

#### 4. 아트보드 B에서 파생 SKETCH TO IMAGE 다중 실행
- **B -> IMAGE_A**: B의 하단으로 생성 (형제 없음 + 파생)
- **B -> IMAGE_B**: IMAGE_A의 하단으로 스태킹 (형제 있음)
```
                            [B:SKETCH]
                                |
                                v
                            [IMAGE_A]
                                |
                                v
                            [IMAGE_B]
```

### 동작 요약
- **첫 아트보드 기준** `SKETCH TO IMAGE` => [우측] 최초 이동 후 [아래로] 스태킹
- **파생(edit) 기준** `SKETCH TO IMAGE` => [아래로] 최초 이동 후 [아래로] 스태킹
- **조작(`edit`) 진입** => [우측]으로 파생 보드 열림
