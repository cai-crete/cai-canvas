## 개요

지우개 모드에서 path(선)가 지워지지 않고 선택/이동되는 버그,
펜 모드에서 기존 객체 위를 그릴 때 선이 그려지지 않고 객체가 선택/이동되는 버그,
SELECT 모드에서 획(path)이 개별 선택/이동되는 버그 (획은 선택 불가여야 함),
그리고 V253에서 연기된 부분 지우기(partial erase) 기능을 구현합니다.

**렌더링 방식**: 픽셀(Canvas) 방식 검토 결과 → 줌 화질 저하·좌표 재구현 등 비용이 크므로
현재 **벡터(SVG) 방식 유지**로 결정.

## 원인 분석

### B-1, B-2 공통 원인

`handlePointerDown` 함수 내 실행 순서 문제:

```
1. pan 모드 체크 → return (정상)
2. resize 핸들 체크 → return (모든 모드에서 실행됨 ← 문제)
3. 아이템 클릭 감지 → 선택+드래그 시작 후 return (모든 모드에서 실행됨 ← 핵심 문제)
4. 모드별 로직: lasso / pen / eraser / text / else
```

**라인 658~700**의 "아이템 클릭 감지" 블록이 캔버스 모드와 무관하게 실행되어,
eraser/pen 모드여도 아이템 위를 클릭하면 선택+드래그가 시작되고 `return`되어
모드별 로직(라인 722+)에 도달하지 못합니다.

### B-3 원인

V253에서 path는 items container 렌더링에서 제외되었으나,
**라인 658~661의 아이템 클릭 감지 `find()`에는 여전히 포함**됩니다.

```typescript
// 현재 코드 — path 포함 (버그)
const clickedItem = [...canvasItems].reverse().find(item =>
  coords.x >= item.x && coords.x <= item.x + item.width &&
  coords.y >= item.y && coords.y <= item.y + item.height
);
```

path 바운딩박스 안을 클릭하면 path가 `clickedItem`으로 감지되어 선택·이동됩니다.
V253 Q&A에서 "path는 선택 불가"로 이미 확인된 사항이나 당시 미처 반영되지 않았습니다.

### F-1 (부분 지우기) 미구현 원인

V253에서 의도적으로 연기됨. 현재 지우개는 `pointerDown` 시 1회 클릭 감지만 하며,
드래그 중 연속 지우기 및 선 분할 로직이 없습니다.

### 변경 2: 지우개 부분 지우기 (Partial Erase)

**2-A: 상태/Ref 추가**

```typescript

```

**2-B: eraseAtPoint 함수 추가**

path 포인트 중 지우개 반경 내에 있는 포인트를 제거하고,
남은 연속 포인트 구간을 새로운 path 아이템으로 분할하여 교체합니다.

```typescript
const eraseAtPoint = useCallback((coords: { x: number; y: number }) => {
  setCanvasItems(prev => {
    const newItems: CanvasItem[] = [];
    let changed = false;

    for (const item of prev) {
      if (item.type !== 'path') {
        newItems.push(item); // path 외 아이템은 무시
        continue;
      }

      const points: { x: number; y: number }[] = item.parameters?.points || [];

      // 지우개 반경 밖의 포인트만 keep
      const keepFlags = points.map(p => {
        const ax = item.x + p.x;
        const ay = item.y + p.y;
        const dist = Math.sqrt((ax - coords.x) ** 2 + (ay - coords.y) ** 2);
        return dist > ERASER_RADIUS;
      });

      const anyErased = keepFlags.some(k => !k);
      if (!anyErased) {
        newItems.push(item); // 이 path는 변경 없음
        continue;
      }

      changed = true;

      // 연속된 keep 구간을 분할하여 새 path 생성
      let segment: { x: number; y: number }[] = [];
      for (let i = 0; i < points.length; i++) {
        if (keepFlags[i]) {
          // 절대 좌표로 변환하여 segment에 추가
          segment.push({ x: item.x + points[i].x, y: item.y + points[i].y });
        } else {
          if (segment.length >= 2) {
            newItems.push(createPathItemFromAbsolutePoints(segment, item));
          }
          segment = [];
        }
      }
      if (segment.length >= 2) {
        newItems.push(createPathItemFromAbsolutePoints(segment, item));
      }
      // segment.length < 2 이면 점/단일점 → 제거
    }

    return changed ? newItems : prev;
  });
}, []);
```

**2-C: createPathItemFromAbsolutePoints 헬퍼 함수**

```typescript
const createPathItemFromAbsolutePoints = (
  absolutePoints: { x: number; y: number }[],
  originalItem: CanvasItem
): CanvasItem => {
  const minX = Math.min(...absolutePoints.map(p => p.x));
  const minY = Math.min(...absolutePoints.map(p => p.y));
  const maxX = Math.max(...absolutePoints.map(p => p.x));
  const maxY = Math.max(...absolutePoints.map(p => p.y));
  const relativePoints = absolutePoints.map(p => ({ x: p.x - minX, y: p.y - minY }));

  return {
    ...originalItem,
    id: `path-${Date.now()}-${Math.random().toString(36).slice(2, 7)}`,
    x: minX,
    y: minY,
    width: Math.max(maxX - minX, 2),
    height: Math.max(maxY - minY, 2),
    parameters: {
      ...originalItem.parameters,
      points: relativePoints,
    },
  };
};
```
