# 작업지시서 V279: 레이아웃 패딩 최적화 및 CHANGE VIEWPOINT 분석(Analyze) 버그 수정

사용자 피드백을 반영하여 오개념을 바로잡고 확정한 V279 최종 전략입니다.

---

## 1. 문제 분석 및 해결 전략 (Strategy)

### [전략 1] 편집(Edit) 버튼 동작 복구 (우측 패딩)
- **요청 사항**: `sketch_generated` 제어바 속 우측 상단 `[+] (edit)` 버튼 클릭 시, 기존 시스템처럼 작동하게 복구.
- **해결 방안**: 
  - 신규 생성 아이템: `type: 'artboard'`, `src: item.src` (기존 이미지 복제본)
  - 위치: `x: item.x + item.width + 120`, `y: item.y` 
  - 이를 통해 사용자가 원본 결과물 바로 우측에서 이어서 스케치를 덧그릴 수 있도록 지원합니다.

### [전략 2] 다중 파생 이미지 세로 정렬 (Fan-out Vertical Stacking)
- **요청 사항**: 동일한 모체(motherId)에서 파생된 이미지들이 우측 공간에 겹치지 않게 배치.
- **해결 방안**: 
  - 파생물이 없을 때: 원본 우측 120px (`x: src.x + src.width + 120`, `y: src.y`)
  - 파생물이 이미 있을 때: 기존 파생 이미지들 중 가장 아래쪽에 있는 이미지(`max(y)`)를 탐색한 후, 그 하단에 120px 간격을 두고 배치 (`x: last.x`, `y: last.y + last.height + 120`).

### [전략 3] CHANGE VIEWPOINT의 분석(Analyze) 재실행 버그 수정
- **현재의 버그 원인 파악**: 
  - 현재 `SKETCH TO IMAGE`를 통해 생성되는 `sketch_generated` 아이템 내부 속성에 하드코딩된 더미 파라미터(`analyzedOpticalParams: { ... }`)가 주입되고 있었습니다.
  - V278에 추가된 자동 분석 방지 로직이 이 더미 파라미터를 보고 "이미 분석이 끝났다"고 착각해 분석(ANALYZE) 단계를 건너뛰고 있었던 현상입니다.
- **해결 방안**:
  1. `sketch_generated` 생성 시 불필요한 `analyzedOpticalParams`, `elevationParams` 등의 더미 데이터를 모두 `null` 처리합니다.
  2. 이를 통해 'CHANGE VIEWPOINT'를 통해 정상적으로 ANALYZE 단계를 거친 이미지가 아니라면, 무조건 새로 분석(ANALYZE)을 시작하도록 조건을 정상화합니다.

---

## 2. 작업 진행 동의 구함

> 오류 증상 코멘트("SKETCH TO IMAGE에서 출력한 이미지는 프로토콜 생성되지 않습니다")가 '제한' 관련 요청이 아닌 **생성되어야 하는데 안 되는 버그(Bug)**를 지적하신 내용이었음을 완벽히 파악했습니다.
> 위 3가지 전략이 맞으시다면, 이 지시서 내용을 토대로 즉시 코드 수정을 진행하겠습니다. 승인 부탁드립니다.
