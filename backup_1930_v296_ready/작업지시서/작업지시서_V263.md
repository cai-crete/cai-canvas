# 작업지시서 V263 — PLANNERS Deep Debate Protocol Integration

**날짜:** 2026-03-28
**작업 범위:** `src/App.tsx` + 신규 lib 파일 3개
**배경:**
현재 cai-canvas의 PLANNERS는 단순 텍스트 프롬프트를 Gemini에 전송하여 하나의 텍스트 응답을 받아 분할 저장합니다.
cai-planners는 **GEMS Protocol 2-Step API**를 통해 전문가 스쿼드를 자동 선정하고, 4인 변증법 토론(Thesis / Antithesis / Synthesis / Support)의 구조화된 JSON 결과를 반환합니다.
본 작업은 cai-planners의 전체 시스템을 cai-canvas에 이식합니다.

---

## 현황 분석

| 항목 | cai-canvas (현재) | cai-planners (목표) |
|------|------------------|-------------------|
| API 호출 | 1회, plain text 반환 | 2회 (squad 선정 + 토론), JSON 반환 |
| 카드 구조 | 흰색 박스 + content 텍스트 | 좌측 전문가 버튼 4개 + 우측 Final Plan 카드 |
| 우측 패널 | "Strategic Output" 텍스트 표시 | 전문가별 상세 발언 (keywords, shortContent, fullContent) + finalOutput |
| 전문가 선정 | 사용자 수동 토글 (1~14번) → 이름만 프롬프트에 삽입 | AI 자동 선정 (Mode 판단 → Squad 배정) |

## 신규 파일 (3개)

### 1. `src/lib/plannerExperts.ts`
cai-planners의 `experts.ts` 전체를 복사합니다.
- 14인 `EXPERTS` 배열 (id, name, personaName, keywords, group, iconName, description, framework)
- `Expert` 인터페이스 export

### 2. `src/lib/plannerSynergyData.ts`
cai-planners의 `synergyData.ts` 전체를 복사합니다.
- `SYNERGY_MATRIX` — 전문가 시너지 조합 우선순위 테이블
- `EXPERT_SUPPORT_MAP` — Support 역할 적합도 가중치
- `EXPERT_PROFILES_COMPACT` — Selector AI용 간략 전문가 프로필
- `MODE_SQUAD_MAP` — Mode A/B/C 별 고정 스쿼드 테이블

### 3. `src/lib/plannerGenerate.ts`
cai-planners의 `gemini.ts`에서 2개 함수를 추출·적용합니다.

```
selectExpertSquad(context, availableIds) → SquadSelection
generateDiscussion(context, mode, selectedExpertIds) → DiscussionResult
```

**API 엔드포인트:** 기존과 동일하게 `https://cai-planners.vercel.app/api/generate` 사용
**모델:**
- Squad 선정: `gemini-flash-latest` (fallback)
- 토론 생성: `gemini-2.5-pro` (fallback: `gemini-flash-latest`)

**반환 타입 `DiscussionResult`:**
```ts
interface ExpertTurnData {
  expertId: string;
  role: 'thesis' | 'antithesis' | 'synthesis' | 'support';
  keywords: string[];
  shortContent: string;
  fullContent: string;
}

interface DiscussionResult {
  metacognitiveDefinition: { selectedMode: string; projectDefinition: string; activeSquadReason: string; };
  workflowSimulationLog: string;
  thesis: ExpertTurnData;
  antithesis: ExpertTurnData;
  synthesis: ExpertTurnData;
  support: ExpertTurnData;
  shortFinalOutput: string;
  finalOutput: string;
  transparencyReport: { selfHealingLog: string; truthfulnessCheck: string; realImpact: string; nextActionSuggestion: string; };
}
```

## App.tsx 변경 사항 (6곳)

### Change 1 — Lucide 아이콘 임포트 추가 (~line 2)
다음 아이콘을 추가로 import합니다 (EXPERTS iconMap 용):

```
Orbit, GitBranch, Shield, Zap, Wind, Hash, History, Target, Cpu, PenTool, Box, Scale
```

- `Search`, `Compass`, `Bot`은 이미 import되어 있음 — 중복 추가 불필요

### Change 2 — CanvasItem 타입 확장 (~line 268)
`parameters` 객체에 `debateResult` 필드 추가:

```ts
// V263: PLANNERS structured debate result
debateResult?: DiscussionResult;
```

`isPlannerResult?: boolean`은 기존 유지 (기존 카드 호환성)

### Change 3 — State 추가 (~line 366)
`selectedPlannerCardId` 상태의 타입·용도 변경:
- 기존: `string | null` — 카드 ID만 저장
- 변경: `string | null` — `"cardId"` 또는 `"cardId::role"` 형식
  - `"cardId"` → 전체 Debate Log 뷰
  - `"cardId::thesis"` 등 → 해당 전문가 단일 뷰

`plannerFeedback` 상태는 제거합니다 (더 이상 사용 안 함).

### Change 4 — GENERATE 로직 교체 (~line 1848~1926)
현재의 단순 텍스트 호출을 2-step 구조로 교체합니다:

```
Step 1: selectExpertSquad(plannerPrompt, allExpertIds)
         → SquadSelection { thesisId, antithesisId, synthesisId, supportId, detectedMode }

Step 2: generateDiscussion(plannerPrompt, detectedMode, allExpertIds)
         → DiscussionResult { thesis, antithesis, synthesis, support, shortFinalOutput, finalOutput, ... }
```

캔버스에 **1개의 카드**만 생성합니다 (기존의 N개 분할 방식 제거):

```ts
const newItem: CanvasItem = {
  id: `planner-${ts}`,
  type: 'text',
  x: cx,
  y: cy,
  width: 540,
  height: 320,
  src: null,
  motherId: null,
  parameters: {
    angleIndex: 0, altitudeIndex: 0, lensIndex: 0, timeIndex: 0,
    content: result.shortFinalOutput,   // 요약 텍스트 (기존 호환)
    fontSize: 13, color: '#1a1a1a',
    isPlannerResult: true,
    debateResult: result                // 구조화 데이터
  }
};
```

`setPlannerFeedback` 제거 (상태 삭제에 따라).
GENERATE 완료 후 패널 상태 변경 없음 — **입력 뷰를 유지**합니다. (B안)
사용자가 캔버스의 카드를 클릭해야 Debate Log 뷰로 전환됩니다.

### Change 5 — 캔버스 카드 렌더링 교체 (~line 2457~2486)
현재의 단순 흰 카드를 TurnGroupNode 스타일로 교체합니다:

**새 카드 레이아웃 (540px, `rounded-[40px] bg-[#EEEEEE] p-3 flex gap-1`):**

```
┌────────────────────────────────────────┐
│ [⚡] [🔍] [🔀] [⚖]  │  FINAL PLAN    │  ← 좌: 전문가 버튼 4개
│  Thesis  Anti  Synth  Supp │  Bot icon  │     우: Final Plan 카드
│                            │  Final     │
│                            │  Strategic │
│                            │  Plan ...  │
└────────────────────────────────────────┘
```

- **좌측 열:** 4개의 `w-11 h-11 rounded-[18px]` 버튼
  - 각 버튼은 해당 전문가의 `iconName`에 매핑된 Lucide 아이콘 표시
  - 클릭: `setSelectedPlannerCardId("cardId::role")` + 패널 열기
  - 선택 상태: `bg-black text-white`, 미선택: `bg-white text-neutral-400`
- **우측 열 (Final Plan 카드):** 기존 흰 카드 유지, `shortFinalOutput` 표시
  - 클릭: `setSelectedPlannerCardId("cardId")` (전체 Debate Log 뷰)

**`debateResult`가 없는 기존 `isPlannerResult` 카드:** 현재 스타일 그대로 유지 (fallback)

### Change 6 — PLANNERS 우측 패널 확장 (~line 3279~3427)
`selectedPlannerCardId`의 값에 따라 3가지 뷰를 렌더링합니다:

## 전문가 패널 UI 변경 (뷰 A 내부)

현재 수동 토글 목록을 **자동 선정 표시 목록**으로 변경합니다.

### 초기 상태 (GENERATE 전)
- 14인 전문가 리스트 전체 표시
- 각 항목: **전문가 아이콘** (iconName → Lucide) + 이름 + 소속 그룹

### GENERATE 완료 후 — 자동 선정 전문가 상위 배치
선정된 4인이 리스트 **상단으로 자동 이동**, 역할 배지 표시:

```
┌──────────────────────────────────────┐
│ [⚡] Roger L. Martin    [THESIS]     │  ← 자동 선정 4인 상단 고정
│ [🔍] Richard P. Rumelt  [ANTITHESIS] │
│ [🔀] Roger L. Martin    [SYNTHESIS]  │
│ [⚖]  Judea Pearl        [SUPPORT]    │
│ ─────────────────────────────────── │
│ [🌐] Henry Mintzberg    조직학자     │  ← 나머지 10인
│ [🛡] Michael E. Porter  경제학자     │
│  ...                                 │
└──────────────────────────────────────┘
```

- 선정 4인: `bg-black/5` 배경 + 역할 배지 (`text-[9px] font-black uppercase tracking-[0.15em]`)
- 나머지 10인: 기존 스타일 유지, 단 토글 불가
- 사용자 수동 토글 제거 (auto-select only)

### 전문가 아이콘 매핑 (Change 1과 연동)
```ts
const IconMap: Record<string, React.ElementType> = {
  Orbit, Search, GitBranch, Shield, Zap, Compass, Wind, Hash,
  History, Target, Cpu, PenTool, Box, Scale, Bot
};
// 사용: IconMap[expert.iconName] ?? Bot
```

### 캔버스 카드 좌측 전문가 버튼 아이콘 (Change 5와 연동)
각 버튼에 해당 전문가의 `iconName` 아이콘 표시:
```tsx
const ExpertIcon = IconMap[expert.iconName] ?? Bot;
<ExpertIcon className="w-[18px] h-[18px]" />
```
