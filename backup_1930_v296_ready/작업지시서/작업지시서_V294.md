## 변경 상세 계획

현재 `src/App.tsx` (약 3122라인 부근)의 `'IMAGE / VIEW'` 라벨의 통합 툴바 버튼 제어 로직을 아래와 같이 교체합니다.

*적용 구문 (`src/App.tsx` 내부 `IMAGE / VIEW` 항목의 `edit(+)` onClick 이벤트)*
```tsx
// 변경 전: setCanvasItems(prev => prev.map(... editVersions 초기화 ...)) 전부 삭제
// 변경 후: 새로운 artboard(CanvasItem 객체)를 만들어서 우측 공간에 배열 삽입(push)

setCanvasItems((prev: CanvasItem[]) => {
  let newX = item.x + item.width + 120;
  let newY = item.y;
  
  // V291 로직 재사용: 이미 다른 파생 보드가 우측에 있다면, 그 뒤로 밀려나게 가로 스캐닝 누적 스태킹
  const siblingEdits = prev.filter((c: CanvasItem) => c.type === 'artboard' && c.motherId === item.id);
  if (siblingEdits.length > 0) {
      const rightMost = siblingEdits.reduce((max, cur) => cur.x > max.x ? cur : max, siblingEdits[0]);
      newX = rightMost.x + rightMost.width + 120;
      newY = rightMost.y;
  }
  
  const newArtboard: CanvasItem = {
    id: `artboard-edit-${Date.now()}`,
    type: 'artboard',
    src: item.src, // 원본 소스의 베이스 이미지 그대로 채용
    x: newX,
    y: newY,
    width: item.width,
    height: item.height,
    motherId: item.id, // V291 앵커 로직을 작동시키기 위해 부모 식별자 연동
    parameters: null
  };

  // 과거 히스토리에 현재 캔버스 상태 저장 (Undo용)
  setHistoryStates((prevH: CanvasItem[][]) => [...prevH, prev]);
  // 새 파생 보드를 캔버스 배열에 부착
  return [...prev, newArtboard]; 
});
```

> **연결선(SVG) 자동 편입**: 이 로직이 교체되면, V291에서 사전 세팅해두었던 SVG 그리기 로직(`자식 타입이 artboard면 무조건 가로선(우측->좌측) 고정`) 조건에 자동으로 부합되어, 별도의 SVG 추가 조작 없이 즉시 예쁜 연결선이 그려집니다.
