## 원인 분석 및 변경 상세 구현

### Change A: SVG 연결선 앵커 고정
기존 연결선(SVG)은 부모와 자식 간의 상대 위치(dx, dy)를 기반으로 `isBelow` 등의 수학을 사용해 상/하/좌/우 면으로 갱신했으나, V289에서는 사용자가 대상을 지맘대로 이동시키더라도 객체의 타입 구조가 가지는 절대 접점에 결속되어야 함.
*적용 구문 (`src/App.tsx` SVG 렌더링 파트)*
```tsx
const isFirstArtboard = !mother.motherId && mother.type === 'artboard';

// 첫 보드인가 파생 보드인가 판단하여 [우측/좌측] 또는 [하단/상단]으로 앵커 확정
const startX = isFirstArtboard ? mother.x + mother.width : mother.x + mother.width / 2;
const startY = isFirstArtboard ? mother.y + mother.height / 2 : mother.y + mother.height;

const endX = isFirstArtboard ? child.x : child.x + child.width / 2;
const endY = isFirstArtboard ? child.y + child.height / 2 : child.y;

// 동적 회전을 삭제하고 곡선 제어점(Control Curve) 거리 고정
const control1X = isFirstArtboard ? startX + 50 : startX;
const control1Y = isFirstArtboard ? startY : startY + 50;

const control2X = isFirstArtboard ? endX - 50 : endX;
const control2Y = isFirstArtboard ? endY : endY - 50;

const pathD = `M ${startX} ${startY} C ${control1X} ${control1Y}, ${control2X} ${control2Y}, ${endX} ${endY}`;
```

### Change B: `edit(+)` 다중 파생 생성 시 (우측) 스태킹 적용
현재 하나의 객체에 `edit(+)`를 여러 번 누르면 기준 좌표 우측에 제자리 생성되어 겹치는 문제가 있지만, 지정된 우측 방향으로 연속 가로 스캐닝을 통해 누적 배치되도록 함.
*적용 구문 (`src/App.tsx` 내부 edit 버튼 핸들러)*
```typescript
setCanvasItems((prev: CanvasItem[]) => {
  let newX = item.x + item.width + 120;
  let newY = item.y;
  
  // motherId와 type 기반으로 이미 생성된 형제 edit 파생 보드가 있는지 확인
  const siblingEdits = prev.filter((c: CanvasItem) => c.type === 'artboard' && c.motherId === item.id);
  
  if (siblingEdits.length > 0) {
      // 형제 보드가 있다면 가장 오른쪽 보드를 찾아 그 옆에 쌓음
      const rightMost = siblingEdits.reduce((max, cur) => cur.x > max.x ? cur : max, siblingEdits[0]);
      newX = rightMost.x + rightMost.width + 120;
      newY = rightMost.y;
  }
  
  const newArtboard: CanvasItem = {
      // ... id, type properties ...
      x: newX,
      y: newY,
  };
  return [...prev, newArtboard];
});
```

### Change C: SVG 연결선 레이어 z-index 강등
현재의 연결선은 이미지가 스태킹되거나 조작될 때 그 위를 가로지르는 문제가 발생함. 전체 캔버스 아이템의 기본 수준인 `z-index: 10`보다 낮춰 렌더링.
*적용 구문 (`src/App.tsx` 캔버스 노드 맵 최상단)*
```tsx
// 기존 z-[100]에서 z-[0]으로 최하위 배경 처리
<svg className="absolute top-0 left-0 w-full h-full pointer-events-none z-[0]">
  ...
</svg>
```
