## Change C: CHANGE VIEWPOINT 업로드/드래그 기능 복원

### 현황 파악 — 두 가지 문제

**문제 1: 캔버스 아이템 드래그 불가**

- canvas 컨테이너 div (line 2124): `onDragStart={(e) => e.preventDefault()}` — 전체 캔버스의 dragstart를 차단
- 캔버스 아이템 div의 `onDragStart` (line 2504)에 `e.stopPropagation()` 없음
- 결과: 아이템 `onDragStart`에서 `text/image-src` 세팅 후 이벤트 버블링 → 캔버스 컨테이너가 `preventDefault()` 호출 → 드래그 취소

**문제 2: 클릭 업로드 기능 없음**

- `SitePlanDiagram` 컴포넌트에 `<input type="file">` 없음
- 오버레이에 `pointer-events-none` → 클릭이 기저 div에 전달되나 파일 선택 트리거 없음

### 변경

**① 캔버스 아이템 `onDragStart` — stopPropagation 추가 (line ~2504)**:
```ts
onDragStart={(e: React.DragEvent) => {
  if ((item.type === 'upload' || item.type === 'generated') && item.src) {
    e.stopPropagation();  // V269: 버블링 차단 (캔버스 컨테이너 preventDefault 우회)
    e.dataTransfer.setData('text/image-src', item.src);
    e.dataTransfer.effectAllowed = 'copy';
  }
}}
```

**② SitePlanDiagram — 파일 입력 추가**:

props, ref, handler 추가:
```ts
const fileInputRef = React.useRef<HTMLInputElement>(null);

const handleFileChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  const file = e.target.files?.[0];
  if (file && file.type.startsWith('image/')) {
    const reader = new FileReader();
    reader.onload = () => onImageDrop?.(reader.result as string);
    reader.readAsDataURL(file);
    e.target.value = '';  // reset for re-upload
  }
};
```

오버레이 변경: `pointer-events-none` → `pointer-events-auto`, onClick 추가:
```tsx
{!sitePlanImage && !isAnalyzing && (
  <div
    className="absolute inset-0 z-20 flex flex-col items-center justify-center cursor-pointer rounded-[inherit]"
    style={{ background: 'rgba(255,255,255,0.65)', backdropFilter: 'blur(6px)' }}
    onClick={() => fileInputRef.current?.click()}
  >
    <Upload size={20} className="opacity-40 mb-2" />
    <span className="font-mono text-[10px] uppercase tracking-widest opacity-40 text-center leading-relaxed">
      UPLOAD<br />or DRAG & DROP
    </span>
    <input
      ref={fileInputRef}
      type="file"
      accept="image/*"
      className="hidden"
      onChange={handleFileChange}
    />
  </div>
)}
```

> 오버레이 클릭 시 파일 피커 열림. 드래그 앤 드롭은 오버레이가 있을 때 클릭 우선, 없을 때(이미지 로드 후) 기저 div의 onDrop 처리.
> 이미지 로드 후 오버레이 사라짐 → 기저 div의 onDrop이 직접 활성화됨.
