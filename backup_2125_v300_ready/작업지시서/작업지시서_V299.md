## 1. 현재 오류 분석 및 근본 원인 추적

### BUG-1: 입면도 5개가 전부 정면(front)과 유사한 이미지로 출력됨

**현상**: ELEVATION SHEET의 REAR, LEFT, TOP, RIGHT, FRONT 5칸에 들어가는 이미지가,
자세히 보면 전부 같은 정면(front) 각도에서 촬영한 것과 거의 동일한 이미지입니다.

**근본 원인**:
현행 `generateOne()` API 호출 구조:

```ts
parts: [
  { inlineData: 원본 이미지 (정면 사진) },  // ← 시각적 우선도 → AI가 이 각도를 강하게 복제
  { text: "Azimuth 180° REAR VIEW..." }     // ← 텍스트는 이미지의 시각적 신호에 밀림
]
config: { temperature: 0.1, topK: 1 }       // ← 극단 결정론 → 원본 이미지 복제를 더욱 강화
```

이미지 생성 AI는 텍스트 명령보다 입력 이미지의 시각적 앵글에 더 강한 영향을 받습니다.
`temperature: 0.1`이 적용될수록 원본 복제가 더 충실해지므로, 5개 모두 정면과 유사하게 생성됩니다.

**[✅ 확정] Q1 — 제안 3 채택: 소스 이미지 미제공**

| 비교 | 제안 1 (프롬프트 강화) | 제안 2 (순차+스택링) | 제안 3 (소스 이미지 미제공) |
|------|----------------------|---------------------|--------------------------|
| 근본 해결 여부 | ❌ 이미지 있는 한 시각적 우선도 문제 잔존 | ❌ 각 호출에 원본 이미지 포함, 속도 5배 저하 | ✅ 시각적 앵커 완전 차단 |
| 재질/형태 정보 | 원본 이미지 참조 | 원본 이미지 참조 | `buildingDescription` + `contextualParamsStr`(AEPL) 텍스트로 충분 |
| 속도 | 병렬 유지 | 5배 저하 | 병렬 유지 |

채택 근거:
- `buildingDescription` (Phase 0 영어 묘사 — 재질/색상/텍스처/조경 포함) + `contextualParamsStr` (AEPL v4 기하학/재질 파라미터) = 텍스트만으로 충분한 참조 정보 확보
- 입면도 목적 = 각도 정확성 > 포토리얼리즘 → 텍스트 기반이 더 적합
- `temperature: 0.1 → 0.4` 상향: 이미지 없이 텍스트 지시만으로 다양한 각도를 생성하려면 변형 여지 확보 필요

### BUG-2: ELEVATION SHEET 아이템의 높이(height)가 16:9를 반영하지 않음

**현상**: ELEVATION SHEET 캔버스 아이템이 실제 16:9 합성 이미지(1536×864)를 담고 있지만,
`height: sourceItem.width` (= 정사각형)로 설정되어 이미지가 세로로 찌그러져 보입니다.

```ts
// 현행 (~line 1724-1725)
width: sourceItem.width,
height: sourceItem.width,   // ← BUG: 1:1 정사각형 가정
```

### Change C — 입면도 생성 프롬프트 수정 (제안 3: 소스 이미지 미제공)

**위치**: `~line 1624–1633` (`generateOne` 내부 API 호출) + prompt 텍스트

#### C-1: API 호출 — 소스 이미지 `inlineData` 제거 + temperature 조정

```ts
// 변경 전
const result = await ai.models.generateContent({
  model: IMAGE_GEN,
  contents: {
    parts: [
      { inlineData: { data: base64Data, mimeType: mimeType } }, // ← 제거
      { text: prompt },
    ],
  },
  config: { temperature: 0.1, topK: 1 }
});

// 변경 후 (V299 C-1: 소스 이미지 미제공 — 텍스트 기반 각도 강제)
const result = await ai.models.generateContent({
  model: IMAGE_GEN,
  contents: {
    parts: [
      { text: prompt },
    ],
  },
  config: { temperature: 0.4 } // topK 제거, temperature 0.1→0.4 (변형 여지 확보)
});
```

#### C-2: prompt 텍스트 — 각도 절대 강제 명령 추가

```ts
// 변경 전 (PHASE 1 첫 줄)
[PHASE 1: GEOMETRY MASTER LOCKING]
- View: ${config.label} (Angle: ${config.angle}, Axis: ${config.normal})
- Camera: Azimuth ${config.az}, Altitude ${config.alt}, Orthographic Projection (FOV=0).
- Snap-to: 100% Geometry guide from Source Image.

// 변경 후 (V299 C-2: 각도 절대 강제 명령 삽입)

CRITICAL: You MUST render the building as viewed from Azimuth ${config.az}° (${config.label} face).

- View: ${config.label} (Angle: ${config.angle}, Axis: ${config.normal})
```
