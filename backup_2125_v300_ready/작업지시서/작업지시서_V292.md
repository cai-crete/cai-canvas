## 원인 분석

### Change A: 연결선이 이미지 위에 렌더링되는 원인

CSS 페인팅 순서 (section stacking context 내):

```
Step 8 : positioned + z-auto (DOM 순서)
           → main artboard area (transform + z-auto) ← 이미지 컨테이너
Step 9 : positioned + z > 0 (z값 순, 동값이면 DOM 순서)
           → 연결선 z-[5]  ← 이미지보다 위에 페인트됨 ✗
           → 오버레이 z-[100~120]
```

main artboard area는 `transform`으로 stacking context를 형성하지만 z-index가 auto이므로 step 8에 배치.
연결선(z-5)은 step 9 → 이미지보다 나중에 페인트 = 이미지 위에 표시됨.

**해결**: 연결선을 `z-[0]`으로 변경 → step 8로 이동.
DOM 순서상 연결선 div(line 2304)가 main artboard div(line 2805)보다 앞에 위치하므로,
step 8 내에서도 이미지보다 먼저 페인트 = 이미지 아래 표시됨 ✓

---

### Change B: edit(+) 파생 아트보드 Upload 버튼 조건

현행 조건 (`~line 3182`):
```tsx
{(!item.label || item.label === 'SKETCH') && (
  // Upload 버튼
)}
```

| 아트보드 종류 | motherId | label | 현재 Upload |
|-------------|----------|-------|-------------|
| 초기 아트보드 | null | undefined → 'SKETCH' | 표시 ✓ |
| sketch_generated [+] 파생 | sketch_generated.id | undefined | **표시됨 ✗** |
| generated [+] 파생 (VIEW XX) | generated.id | undefined | **표시됨 ✗** |

파생 아트보드(`motherId !== null`)는 부모 이미지(src)를 그대로 사용하므로 Upload 불필요.

**해결**: `&& !item.motherId` 조건 추가 → motherId 있는 artboard에서 Upload 숨김.
