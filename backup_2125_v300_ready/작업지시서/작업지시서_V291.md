**[2026-03-31 사용자 1, 2차 확정 사항 요약]**
1. **기타 AI 생성 결과물의 유지 여부 (Q1 제안 1 선택)**
   - `edit(+)` 버튼을 통해 명시적으로 생성된 파생 보드(artboard)에 대해서는 연결선 접점을 **[가로 방향(좌측 ↔ 우측)]**으로 편입시킵니다.
   - 단, `SKETCH TO IMAGE` 등 다른 생성 결과물 스케치 타일은 기존 세로(하단 ↔ 상단) 방식을 유지합니다.
2. **`CHANGE VIEWPOINT` (VIEW XX) 레이아웃 및 앵커 기준 (Q2 응답 완벽 반영)**
   - 레이아웃(배치): 첫 번째 이미지(VIEW 01)는 원본 이미지의 우측에 생성되며, 두 번째 이미지(VIEW 02)부터는 첫 번째 이미지(VIEW 01) 바로 밑(하단)으로 차곡차곡 누적(스태킹)됩니다.
   - 연결선(앵커): 이미지들이 쌓이는 위치 변화와 무관하게, **모든 VIEW XX 생성 이미지로 향하는 연결선은 가로 방향(원본 중앙 우측 ↔ 파생 보드 중앙 좌측)으로 일괄 고정**합니다.

## 변경 상세 계획

### Change A & B: SVG 연결선 방향 판별 조건식 최종 병합
현재 `App.tsx`의 연결선 렌더링 조건을 업데이트하여, `edit(+)` 파생 보드(`artboard`)와 `CHANGE VIEWPOINT` 파생 이미지(`generated`)를 모두 가로 연결선의 기본 대상으로 포함합니다.

*적용 예정 구문 (`src/App.tsx` 연쇄 연결선 렌더링 파트)*
```tsx
const isFirstArtboard = !mother.motherId && mother.type === 'artboard';
const isEditDerived = child.type === 'artboard';
const isViewpoint = child.type === 'generated';

// 가로 방향(원본 좌측 -> 자식 우측 스크롤) 선을 고정해야 하는 경우: 
// 1) 첫 메인 보드의 자식 이거나 2) edit 파생 보드 이거나 3) CHANGE VIEWPOINT 결과물일 때
const isHorizontalConnection = isFirstArtboard || isEditDerived || isViewpoint;

const startX = isHorizontalConnection ? mother.x + mother.width : mother.x + mother.width / 2;
const startY = isHorizontalConnection ? mother.y + mother.height / 2 : mother.y + mother.height;

const endX = isHorizontalConnection ? child.x : child.x + child.width / 2;
const endY = isHorizontalConnection ? child.y + child.height / 2 : child.y;

// 대각선/직선 제각각인 타일에 매끄러운 S자 베지어 곡선을 주기 위한 제어점(Control Point)
const control1X = isHorizontalConnection ? startX + 50 : startX;
const control1Y = isHorizontalConnection ? startY : startY + 50;
const control2X = isHorizontalConnection ? endX - 50 : endX;
const control2Y = isHorizontalConnection ? endY : endY - 50;

const pathD = `M ${startX} ${startY} C ${control1X} ${control1Y}, ${control2X} ${control2Y}, ${endX} ${endY}`;
```
