## 질문

### Q1. CHANGE VIEWPOINT 'ANALYZE' — "이미지 재생성" 의미

ANALYZE 버튼 클릭 후 동작 중 "이미지 재생성 + PHASE 2 발동"에서 **이미지 재생성**이 무엇인지 불명확합니다.

**방안 1** — ANALYZE = 기존 `analyzeViewpoint()` 실행만 (파라미터 분석 + elevation 이미지 생성)
- "이미지 재생성" 없음 — 원본 이미지 그대로 유지
- PHASE 2 = 슬라이더 활성화 + GENERATE 버튼 노출

**방안 2** — ANALYZE = `analyzeViewpoint()` + Gemini로 원본 이미지 재생성
- 원본 이미지를 분석하여 "ANALYZED" 레이블로 재생성된 이미지 탭 추가
- PHASE 2 = 재생성된 이미지를 기반으로 CHANGE VIEWPOINT GENERATE 진행

**방안 3** — ANALYZE = `analyzeViewpoint()` 실행
- "재생성"은 ANALYZE 단계가 아닌, 이후 GENERATE 단계를 의미
- PHASE 2 = GENERATE 버튼으로 VIEWPOINT 이미지 생성

### Q3. [>] 클릭 시 새 EDIT 버전 생성 방식

**방안 1** — [>] 클릭 → 우측 패널 열림 → 파라미터 수정 후 GENERATE 버튼으로 새 탭 추가
- 가장 유연, 사용자가 수동으로 편집

**방안 2** — [>] 클릭 → 현재 아이템의 파라미터로 자동 재생성 (동일 설정으로 variation)
- 클릭 즉시 Gemini 호출 → 새 이미지 자동으로 탭에 추가

**방안 3** — [>] 클릭 → "이 이미지를 수정하시겠습니까?" 확인 후 해당 패널 열림
- 확인 다이얼로그 → 패널 열림 → 파라미터 조정 → GENERATE
