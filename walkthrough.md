# Walkthrough: 로더 UI 개편 및 Z-Index 계층 동기화 구축 (V282)

이전 버전의 기획들(V280~V281)을 이어받아 시각적 로더(로딩 상태 표시기)의 계층을 정립하고 레이블 롤백을 수행한 V282 업데이트 결산입니다.

## 1. 개별 로더 그래픽 Z-index 계층 교정 (사용자 직접 반영 검토)
생성 및 분석 서버와 비동기 통신을 할 때 화면에 등장하는 스피너 로더가 다른 객체들(스케치나 선택 테두리)과 겹칠 때 발생하는 Layer 간섭 문제가 구조적으로 보완되었습니다.

- **독립 오버레이 (`z-[102]`) 구축**: 로더 요소가 각 캔버스 아이템의 내부 `div` 구조에서 벗어나, 캔버스 전역을 오버레이하는 가장 상단 영역으로 분리 편입되었습니다. 
- **Z-index 계위 완성**:
  1. `[최상단] 선택 시 뷰 컨트롤(파란테두리 및 조절점)`: `z-[105]`
  2. `[로더] 분석/생성 스피너 반투명 오버레이`: `z-[102]`
  3. `[스케치] 사용자가 브러시/펜으로 그은 선(path)`: `z-[100]`
  4. `[최하단] 실제 업로드/생성 기본 이미지`
- 위 서열에 따라, 스케치 선을 마구 그어둔 캔버스 위에서 CHANGE VIEWPOINT를 실행하여 로더가 돌면, 스케치 선 위로 반투명 로딩 막이 예쁘게 덮여 가려지게 됩니다.

## 2. 레이블링(Naming) 룰 원복 및 Book 아이콘 동기화
- **`EDITED 01` 원복**: 이전 패치에서 변경되었던 생성물 레이블을, 사용자의 의도에 따라 다시 모체(EDIT XX) 번호를 물려받는 **`EDITED 01~`** 규칙으로 롤백(유지)했습니다.
- **도서관(Book) 버튼 누락 해결**: ANALYSIS 완료된 상태의 모체(`label: ANALYZED`)의 제어바에 입면 전개도 Book 버튼을 노출시킬 때, 그것이 업로드 이미지이든 혹은 순수 아트보드이든 상관없이 모두 렌더링되도록 조건문을 완벽히 호환시켰습니다.

## 💡 최종 확인

1. CHANGE VIEWPOINT를 눌렀을 때, 로더가 현재 대상 캔버스 위를 잘 덮어씌우는가? 
2. 로더가 도는 중일 때 그려져 있던 스케치 선 위로 반투명하게 가려지는가? (뷰 컨트롤 테두리보단 아래인가?)
3. 빈 캔버스 기반이든 업로드 기반이든 상관없이 ANALYZED 후 상단 제어바에 Book 아이콘이 노출되는가?
4. SKETCH 결과 파생물 배너가 `EDITED 01` 등으로 잘 표시되는가?
