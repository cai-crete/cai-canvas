## Change 6 — 스케치 선 / 연결선 가시성 수정

**현황 파악:**
- Connection SVG: `zIndex: 0` (V264 적용) — 캔버스 아이템 위에 표시되는 문제
- 문제 가능 원인: 캔버스 아이템 wrapper의 `zIndex` (선택 시 20, 미선택 시 10) vs SVG `zIndex: 0`

**분석:**
- 캔버스 아이템 div는 CSS `position: absolute`와 `zIndex: 10~20`
- Connection SVG도 `position: absolute`이며 `zIndex: 0`
- 그러나 SVG는 캔버스 아이템 **이후에** DOM에 배치되어 stacking context에서 위에 올라올 수 있음

**해결 방향:**
- Connection SVG를 DOM 순서상 **캔버스 아이템 이전**으로 이동
- 또는 `zIndex: -1`로 변경 (단, 부모 stacking context 주의)
