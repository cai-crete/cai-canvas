**작성일**: 2026-04-01
**버전**: V312 (V310.1 + V311 통합본)
**상태**: 완료 (반영 완료)

---

## 1. 개요 및 목표
V310.1의 스케치 인터렉션 개선 사항과 V311의 상단 제어바 UI 최적화 요구사항을 통합하여 최종 반영하였습니다. 사용자 경험(UX) 측면에서 불필요한 클릭을 줄이고, 스케치 작업 시 의도치 않은 배경 낙서를 방지하며, 객체 조작의 유연성을 확보하는 것을 목표로 합니다.

## 2. 주요 업데이트 내용

### ✅ 1. 스케치 인터렉션 정교화
*   **이미지 위 스케치 허용**: 이미지 객체(`upload`, `generated`) 클릭 시 UI 인터렉션 블록에 의해 스케치가 시작되지 않던 문제를 `data-canvas-item` 속성 및 PointerDown 예외 처리를 통해 해결했습니다. 이제 이미지 위에서 막힘없이 즉시 스케치가 가능합니다.
*   **캔버스 스케치 차단**: 스케치 유효 대상을 이미지 영역으로 한정했습니다. 빈 공간이나 배경 도화지(`artboard`) 클릭 시에는 자동으로 화면 이동(`pan`) 모드로 작동하여 실수를 방지합니다.
*   **영역 내 드로잉 제한**: 선을 그리는 도중 이미지 경계 밖으로 마우스가 나가더라도, 시작된 이미지 내부에만 스케치가 남도록 좌표 범위를 엄격히 제한합니다.

### ✅ 2. 드래그 조작성 유지
*   **커서 모드 이동 자유화**: V310에서 기획되었던 고정 모드 대신, 사용자의 요구에 따라 `select` 모드에서 파생 이미지를 자유롭게 드래그하여 배치할 수 있는 기능을 그대로 유지 및 복구했습니다.

### ✅ 3. UI 지능형 자동화 (+ 버튼 제거)
*   **중복 진입 방지**: 파생 이미지 내부의 `+ 버튼`을 클릭하여 이미 편집 가능 상태(`upload` 타입 및 `motherId` 보유)가 된 경우, 상단 제어바에서 `+ 버튼`을 즉시 제거합니다. 이를 통해 UI를 간결하게 유지하고 동일 객체에 대한 중복 생성 탭 누적을 방지합니다.
