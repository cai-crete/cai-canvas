## 발견된 버그 3건

### BUG-1: 펜툴 드로잉 시 획이 2곳에 렌더링됨

**원인**
완성된 path 아이템이 두 곳에서 동시에 렌더링됨:

| 위치 | 코드 | 좌표계 |
|------|------|--------|
| SVG 오버레이 레이어 (line ~1748) | `canvasItems.filter(i => i.type === 'path').map(...)` | 상대 좌표(relativePoints)를 절대 캔버스 좌표로 오해하여 잘못된 위치에 렌더링 |
| Items 컨테이너 (line ~2016) | `item.type === 'path'` 분기에서 인라인 SVG 렌더링 | viewBox 기반 상대 좌표, 올바른 위치 |

**추가 오류**: SVG 오버레이의 path 렌더링은 `relativePoints`(minX/minY 기준 상대값)를 캔버스 절대 좌표로 사용하므로, items 컨테이너와 다른 위치에 렌더링됨.

**수정 방향**
- "선은 객체가 아닌 선으로 생성되어야 한다"는 요구에 따라, **path는 SVG 오버레이 레이어에서만 렌더링**
- SVG 오버레이의 path 렌더링 수식을 `(item.x + p.x, item.y + p.y)` (절대 캔버스 좌표 변환)으로 수정
- **Items 컨테이너에서 `type === 'path'` 아이템 렌더링을 완전히 제거** (outer div 자체를 skip)
- `canvasItems`에는 계속 저장 (Undo/Redo, 지우개 동작 유지)

### BUG-2: 텍스트 박스 생성 후 선택 상태가 시각적으로 표시되지 않는 문제

**현재 코드 흐름 분석**
`handlePointerUp` (line ~903–904):
```js
setCanvasItems(prev => [...prev, newItem]);
setSelectedItemIds([newItem.id]);   // ← 호출됨
```
상태는 설정되나 `selectedItemIdsRef.current`가 업데이트되지 않음.
→ 시각적 선택 테두리는 state 기반이므로 렌더링 자체는 되어야 함.

**실제 문제**: 텍스트 박스가 선택된 것처럼 보여도 이후 즉시 다른 상호작용이 선택을 해제할 가능성.
추가 확인 필요: `setSelectedItemIds` 호출 이후 handlePointerUp 내에서 다른 `setSelectedItemIds([])`가 실행되는지.

**수정 방향**
- 텍스트 아이템 생성 직후 `selectedItemIdsRef.current`도 동기적으로 업데이트:
  ```js
  setSelectedItemIds([newItem.id]);
  selectedItemIdsRef.current = [newItem.id];  // ← 추가
  ```

### BUG-3: SELECT 모드에서 텍스트 박스를 클릭해도 선택되지 않음

**원인**
텍스트 박스 내부 `<textarea>`에 `onPointerDown={(e) => e.stopPropagation()}` 처리 (line ~2013).
`<textarea>`가 아이템 전체 영역(`w-full h-full`)을 커버하므로,
SELECT 모드에서 텍스트 박스를 클릭해도 캔버스 `handlePointerDown`에 이벤트가 전달되지 않음.
→ 캔버스의 bounding-box 기반 아이템 감지 로직이 실행되지 않아 선택 불가.

**수정 방향**
텍스트 아이템에 한해, SELECT 모드일 때 `<textarea>` 위를 덮는 투명한 포인터 캡처 레이어를 추가:

```jsx
{canvasMode === 'select' && (
  <div
    className="absolute inset-0"
    style={{ zIndex: 10, pointerEvents: 'auto' }}
  />
)}
```

이 div는 이벤트를 stopPropagation 없이 버블업시켜 캔버스 `handlePointerDown`에 전달.
캔버스는 bounding-box 검사로 해당 텍스트 아이템을 감지하여 선택.
