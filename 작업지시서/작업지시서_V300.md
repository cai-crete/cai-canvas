## 1. 현황 분석 및 확정 방향 (Q1 제안 2 채택)

### V299 C-1 부작용 — 형태 정확도 하락

V299에서 각도 고착 문제를 해결하기 위해 **소스 이미지를 완전히 제거**하고 텍스트로만 렌더링한 결과:
- ✅ 각도는 전후좌우 다양하게 잘 돌아감
- ❌ AI가 원본 건물의 복잡한 3D 유기적 형태와 디테일을 잃어버리고 완전히 다른 건물을 임의 생성함

### [✅ 확정] Q1 제안 2: FRONT 우선 생성 후 시점 스택링 (Sequential Stacking)

원본 이미지의 '형태 앵커' 역할을 복원하되 각도 고착을 피하기 위해 **2-Step 순차 생성 방식**을 채택합니다.

**작동 원리**:
1. **Step 1 (FRONT 생성)**: 원본 이미지를 그대로 제공하여 형태와 재질을 100% 복제하는 FRONT 뷰를 먼저 생성합니다.
2. **Step 2 (나머지 4개 뷰 생성)**: 원본 이미지 1장만 주면 AI가 다른 각도로 돌리기 힘들어하므로, **"원본 이미지 + 방금 생성한 FRONT 이미지"** 2장을 동시에 제공합니다. 
   - 이로 인해 "이미 렌더링 된 3D 공간 규칙"이 컨텍스트에 쌓여(Stacking), 나머지 `RIGHT, LEFT, REAR, TOP` 명령을 내릴 때 AI가 형태를 유지하면서 각도만 성공적으로 돌릴 수 있게 됩니다.

## 2. 코드 변경 계획 (`src/App.tsx`)

```ts
// 변경 전 (~1578)

  // ... split 로직 부재
}

// 변경 후 (V300 A: 소스 이미지 파라미터 완전 복원)
const generateElevations = async (base64Image: string, extractedParams?: any, itemId?: string, buildingDescription: string = '') => {
  // ...
  const base64Data = base64Image.split(',')[1];
  const mimeType = base64Image.split(';')[0].split(':')[1];
```

### Change B: 2-Step 순차 스택링 로직 구현

기존의 단순 병렬 `Promise.all` 생성 방식을 2-Step으로 쪼갭니다.

**Step 1: FRONT 전용 생성**
```ts
// frontConfig 추출 후 단일 실행
const frontConfig = viewConfigs.find(c => c.key === 'front')!;
// FRONT는 원본 이미지 1장 반환
const frontBase64 = await generateOne(frontConfig, [{ inlineData: { data: base64Data, mimeType } }]);
```

**Step 2: 나머지 4개 프롬프트 수정 및 병렬 생성**
FRONT 생성 완료 후, `top, right, left, rear` 4개는 `parts` 배열에 원본 이미지 + FRONT 이미지를 둘 다 탑재하여 병렬 생성합니다.

```ts
const otherConfigs = viewConfigs.filter(c => c.key !== 'front');

// 프롬프트 강제: "IMAGE 1 = 소스 (재질), IMAGE 2 = FRONT 뷰 (형태 기준)"
const generateStacked = async (config) => {
  // ...
  // 온전한 형태 앵커를 위해 온도 0.1 복구
  const result = await ai.models.generateContent({
    model: IMAGE_GEN,
    contents: {
      parts: [
        { text: "IMAGE 1: SOURCE CONTEXT" },
        { inlineData: { data: base64Data, mimeType } },
        { text: "IMAGE 2: FRONT VIEW REFERENCE (ANCHOR)" },
        { inlineData: { data: frontBase64.split(',')[1], mimeType: 'image/png' } },
        { text: prompt } // "CRITICAL: MUST Rotate to Azimuth..." 텍스트 포함
      ]
    },
    config: { temperature: 0.1 } 
  });
  // ...
};

const otherResults = await Promise.all(otherConfigs.map(generateStacked));
```

### Change C: `prompt` 내용 재구성 (이미지 스택 인식)

Step 2에서 넘겨주는 프롬프트 텍스트를 위 2장의 이미지 체계에 맞게 업데이트합니다.

```ts
// V300 C: 스택링 프롬프트
[PROTOCOL B: 2-STAGE STACKING ENGINE]
IMAGE 1 (Source) is for Material and Style ONLY.
IMAGE 2 (Front View) is your 3D Geometry Anchor.
CRITICAL: You are now flying the camera to Azimuth ${config.az}° to establish the ${config.label} face.
Do NOT reproduce the front face. Rotate the subject correctly.
```
