# 작업지시서 V308 (파생 이미지 독립화 UI 및 로컬 스토리지 최적화)

**작성일**: 2026-04-01
**버전**: V308
**상태**: 승인 대기 (수행 계획 제안)

---

## 1. 개요 및 목표
사용자 피드백에 따라 파생 이미지(`sketch_generated` 및 `generated`) 상단의 `+` 버튼 기능에서 불필요하게 모체와의 연결선이 끊기거나 버전 라벨 좌우에 꺾쇠 화살표(`<`, `>`)가 생기는 현상을 방지하고, 이를 통해 스케치가 가능한 상태로 올바르게 전환하도록 로직을 수정합니다.
또한, 이미지가 누적될 때마다 발생하는 `QuotaExceededError` (로컬 스토리지 5MB 용량 초과) 버그를 방지하기 위해 IndexedDB를 도입하여 `crete_canvasItems` 상태 저장 방식을 최적화합니다.

## 2. 작업 상세 내용 (수행 계획)

### 🚩 파생 이미지 상단 `+` 버튼 로직 개선
*   **문제**: 기존 코드에서는 `+` 버튼 클릭 시 `motherId`를 강제로 `null`로 초기화하여 선이 끊기고, `editVersions` 배열을 생성하여 `<` `>` 화살표 라벨을 노출시켰습니다. (또는 `generated`의 경우 자신을 삭제하고 모체를 대체하는 등 시각적 흐름 파괴).
*   **해결**:
    *   버튼 클릭 시 `type: 'upload'` 상태로 넘어가서 스케치/편집이 가능하도록 승격시킵니다.
    *   단, **`motherId`를 기존대로 유지**하여 모체와의 연결선(Bezier Curve)이 끊어지지 않도록 보호합니다.
    *   **`editVersions` 필드를 삽입하지 않음**으로써 하단 네이밍 라벨 좌우에 `< >` 기호가 발생하지 않도록 제어합니다.
    *   시각적 흐름상 이미지가 우측으로 추가되는 방식이 아니라 제자리에서 상태만 편집 가능하게 바뀌도록 구현하여 통일감을 유지합니다.

### 🚩 로컬 스토리지 QuotaExceededError 해결
*   **현상**: `App.tsx`에서 모든 `canvasItems` 데이터(거대한 Base64 `src` 문자열 포함)를 `localStorage.setItem('crete_canvasItems', ...)`로 매번 저장하여, 브라우저 스토리지 5MB 할당량을 초과하면 오류를 뿜고 강제 중단되는 치명적 버그가 존재합니다.
*   **해결**: 
    *   방치되어 있던 `useCanvasStore.ts`의 `saveImageToDB` 및 `loadImageFromDB` 헬퍼 함수를 `App.tsx`에 가져와 연동합니다.
    *   앱 최초 실행(`useEffect` 마운트) 시 IndexedDB에서 `src` 데이터를 비동기적으로 Load 합니다.
    *   사용자가 이미지를 추가/수정하여 `canvasItems`가 갱신될 때마다 **`src`는 IndexedDB에만 저장**하고, `localStorage`에는 `src: ''`처럼 내용물을 비워서 JSON 형태로 안전하고 가볍게 저장하도록 개선합니다.
