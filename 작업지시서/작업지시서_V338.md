# 작업지시서 V338: CHANGE VIEWPOINT UI 및 제어 최적화

## 1. 작업 개요
CHANGE VIEWPOINT 패널 및 상단 제어바의 UI를 단순화하고 사용성을 개선합니다. 분석 결과 텍스트 간소화, 불필요한 컨트롤 제거, 스크롤 영역 제한, 고정된 제어 값 제공을 목표로 합니다.

## 2. 주요 변경 사항

### 2.1. ANALYSIS 섹션 간소화
- **ALTITUDE, LENS 텍스트 간소화**: Gemini API에서 반환되는 복잡한 설명 대신 숫자로만 표기되도록 파싱하거나 렌더링 로직을 변경합니다. (예: `150m`, `35mm`)

### 2.2. RE-ANALYZE 버튼 이동
- 패널 내의 `RE-ANALYZE` 버튼을 제거합니다.
- 모체 이미지 상단 제어바 가장 왼쪽에 새로고침 아이콘(🔄)을 추가하여 재분석 기능을 담당하도록 합니다. (단, 해당 버튼은 이미지 분석이 한 번이라도 완료된 상태(`cvHasAnalyzed === true`)인 경우에만 나타납니다.)
- 상단 제어바 아이콘 순서: `[새로고침] | [+] | [다운로드] | [휴지통]`

### 2.3. 레이아웃 및 제어부 단순화
- **전체 스크롤 폐기**: 패널 전체가 스크롤되지 않도록 고정합니다.
- **VIEW 탭 내부 스크롤**: 내용이 흘러넘치는 경우 VIEW 섹션 안에서만 스크롤이 적용되도록 CSS를 수정합니다.
- **ADDITIONAL PROMPT 제거**: 추가 프롬프트 입력창을 삭제합니다.

### 2.4. 뷰 세부 옵션 UI 변경
- **DIRECTION 버튼 간소화**: 방향 버튼에서 텍스트(RIGHT, LEFT, (R), (L) 등)를 완전히 제거하고 시간만 남깁니다. (예: `03:00`, `09:00`, `04:30`, `07:30`)
- **ALTITUDE 단계별 적용**: 고도 조절 input을 슬라이더에서 5단계(`0m`, `1.6m`, `10m`, `50m`, `150m`)를 선택할 수 있는 방식으로 변경합니다. 

## 3. 진행 절차
1. 본 작업지시서(V338) 확인 및 승인.
2. `App.tsx` 코드 리팩토링.
    - 패널 CSS 수정 (스크롤 분리).
    - 분석 값 파싱/렌더링 로직 수정.
    - 상단 제어바 아이콘 추가.
    - 추가 프롬프트 텍스트 영역 제거 및 방향/고도 제어부 수정.
3. 작업 완료 후 동작 검증 및 V338 마무리.
