# 작업지시서 V306 (UX 고도화 및 제어권 독립)

**작성일**: 2026-04-01
**버전**: V306
**상태**: 승인 대기 (수행 계획 제안)

---

## 1. 개요 및 목표
사용자 테스트 피드백에 따라 기존 V305의 아쉬운 점 5가지를 즉각 수리하고, UI/UX의 직관성과 파생 이미지의 독립성을 강화합니다.

## 2. 작업 상세 내용 (수행 계획)

### 🚩 1. 빈 캔버스 클릭 시 패널 초기 상태(업로드 직후 상태)로 복귀
*   **현상**: V305에서는 캔버스 빈 곳을 누르면 우측 패널 전체가 아예 사라지도록(Closed) 만들었습니다.
*   **해결**: 패널이 완전히 닫히는 대신, **'SELECT TOOLS' (기능 선택 탭) 목록이 열려 있는 기본 상태**로 복귀하도록 수정합니다. (이미지를 방금 업로드한 직후의 우측 패널 상태)

### 🚩 2. 'edit(+)' 파생 이미지 독립 모체화 (제자리 교체)
*   **현상**: V305에서는 파생 이미지에서 `+`를 누르면 너무 멀리 떨어진 **'원본 모체'** 위치로 날아가 덮어쓰기가 되었습니다.
*   **해결**: `+` 버튼의 **주인(현재 클릭한 파생 이미지)** 자체가 제자리에서 '새로운 독립 모체'로 승격(교체)되도록 수정합니다. 즉, 캔버스 상의 자기 위치를 그대로 유지한 채 `motherId` 종속성을 끊어버려, 스스로가 새로운 변화의 시작점(Mother)이 되게 만듭니다.

### 🚩 3. GENERATE 도중 선택(Selection) 락인 (Lock-in)
*   **현상**: 생성 중일 때 실수로 빈 캔버스를 누르면 이미지가 선택 해제되어 진행 상황 파악이 어렵고 우측 패널도 꺼집니다.
*   **해결**: 시스템이 `isGenerating` (또는 `isAnalyzing`) 상태일 때는 **빈 캔버스를 클릭해도 현재 선택된 이미지가 해제되지 않도록 강제로 고정(Lock)**합니다. 작업이 완료될 때까지 대상 이미지의 선택 상태와 우측 패널을 안전하게 보호합니다.

### 🚩 4. IMAGE TO ELEVATION 실행 시 우측 패널 전환 방지
*   **현상**: `IMAGE TO ELEVATION` 버튼을 누르면 무조건 우측 패널이 해당 기능의 세부 패널로 빨려 들어가듯 전환되었습니다.
*   **해결**: 해당 기능은 캔버스 상에 5면도를 알아서 흩뿌려주는 백그라운드 성격이 강하므로, 버튼을 눌러도 우측 패널이 **'SELECT TOOLS' 목록 상태에 그대로 머물도록** (패널 전환이 일어나지 않도록) 제어합니다.

### 🚩 5. 캔버스 로더 하단에 'CANCEL' 알약 버튼 추가
*   **현상**: 현재 캔버스 상의 로더(빙글빙글 도는 아이콘)에는 작업을 취소할 수 있는 직관적인 버튼이 위젯 근처에 없습니다.
*   **해결**: 캔버스에 렌더링되는 로더 그래픽 수직 하단(12px 패딩)에 **단독 [CANCEL] 버튼**을 띄웁니다.
    *   **스타일**: 폰트 크기 `12px` (우측 패널과 동일한 `font-mono`), 알약 모양 (`rounded-full`), 버튼 안쪽 텍스트 상하좌우 `3px` 패딩.
    *   **기능**: 클릭 시 진행 중인 분석/생성 프로세스 전면 취소 (`handleCancelGenerate` 트리거).
