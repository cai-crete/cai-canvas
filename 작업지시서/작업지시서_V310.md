# 작업지시서 V310 (스케치보드 고정 및 상태 동기화 최종 수정)

**작성일**: 2026-04-01
**버전**: V310
**상태**: 승인 대기 (수행 계획 제안)

---

## 1. 개요 및 목표
V309에서 적용된 파생 이미지 스케치 활성화 로직 중, 이미지가 여전히 드래그되는 현상과 스케치가 즉시 원활하지 않은 문제를 해결합니다. 기존(V294 이전)에 우측에 파생되던 "편집 탭"의 속성을 분석하여 이를 현재의 "제자리 편집" 환경에 맞게 이식하고, 브라우저 기본 드래그 속성까지 완벽히 차단하여 "고정된 스케치보드"로서의 기능을 완성합니다.

## 2. 조사 내용: 기존 파생 이미지의 상태 (V294 기준)
기본적으로 `+` 버튼 클릭 시 생성되던 파생 객체는 다음과 같은 상태를 가졌습니다:
- **Type**: `artboard` (단순 `upload` 이미지가 아닌 편집용 판넬)
- **Visual**: 배경에 `rgba(248, 244, 232, 0.3)` 색상과 블러 필터가 적용되어 "도화지" 느낌을 줌.
- **MotherId**: 원본 이미지의 ID를 참조하여 연결선 유지.
- **Parameters**: `null`로 초기화되어 기존 생성 파라미터(Debate 등)의 간섭을 배제.

## 3. 작업 상세 내용 (수행 계획)

### 🚩 1. 파생 승격 타입을 'artboard'로 변경
*   현재 `upload` 타입으로 변경되던 것을 기존 아키텍처에 맞게 `artboard` 타입으로 변경합니다.
*   이는 시각적으로 "편집 가능한 도화지" 상태임을 명확히 하고, `upload` 타입에 걸려있는 기본 드래그 속성과의 충돌을 피하기 위함입니다.

### 🚩 2. `handlePointerDown` 드래그 차단 로직 수정
*   **오류 수정**: V309에서 `motherId !== null`로 체크하던 로직을 `!!motherId` (존재 여부)로 수정하여 `undefined` 값에 의한 오판단을 제거합니다.
*   **대상 확장**: 파생된 `artboard` 및 `upload` 아이템 중 `motherId`가 있는 모든 객체는 드래그 이동이 불가능하도록 명시적으로 잠급니다.

### 🚩 3. JSX `draggable` 속성 동적 제어
*   이미지 컨테이너의 HTML `draggable` 속성이 `upload` 타입일 때 무조건 `true`로 설정되어 브라우저 고스트 이미지가 생기는 문제를 수정합니다.
*   `draggable={item.type === 'upload' && !item.motherId}` 와 같이 수정하여, 파생되어 고정되어야 할 이미지에서는 브라우저 기본 드래그 기능도 작동하지 않게 합니다.

### 🚩 4. 스케치 모드 즉시 진입 유지
*   버튼 클릭 시 `canvasMode`를 `pen`으로 전환하는 로직을 유지하여, 클릭 즉시 그림을 그릴 수 있는 인터렉션을 보장합니다.

기존 시스템의 `artboard` 상태를 재활용하여 사용자가 원하시는 "움직이지 않는 스케치 도화지" 환경을 완벽히 구축하겠습니다.
위 계획이 타당하다면 승인을 부탁드립니다. 승인 시 즉시 반영하겠습니다.
