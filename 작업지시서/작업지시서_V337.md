# 작업지시서 V337: CHANGE VIEWPOINT 초기 UI 통합 및 레이아웃 개선

## 1. 작업 개요
현재 CHANGE VIEWPOINT 패널의 초기 진입 시 빈 화면(안내 아이콘) 대신, 분석 전에도 전체 구조(VIEWPOINT, ANALYSIS, VIEW)가 보이도록 UI를 통합하고 사용자 제공 이미지 디자인을 반영합니다.

## 2. 주요 변경 사항

### 2.1. 패널 레이아웃 통합 및 섹션화
- 기존 Stage 1(분석 전) / Stage 2(분석 후)로 나뉜 구조를 **하나의 통합 레이아웃**으로 변경합니다.
- 패널은 다음 세 가지 섹션으로 구성됩니다:
    - **VIEWPOINT**: `SitePlanDiagram` 상시 노출. **(분석 완료 후, 해당 뷰어 하단 점(dot) 표시나 시계판 각도 등으로 도출된 Angle 값을 시각적으로 연동 표기)**
    - **ANALYSIS**: 분석 데이터(Angle, Altitude, Lens) 리스트. 분석 전에는 값을 `none`으로 표시.
    - **VIEW**: 5가지 뷰 선택 버튼 미리 노출.
- 하단 저작권 문구(`© CRETE CO.,LTD. 2026. ALL RIGHTS RESERVED.`) 및 버튼 스타일은 유지합니다.

### 2.2. 뷰 버튼(VIEW) 레이블 및 순서 변경
- 사용자 이미지에 맞춰 레이블을 다음과 같이 수정합니다:
    - **BIRD'S EYE VIEW**
    - **PERSPECTIVE VIEW** (기존 AISLE LEVEL VIEW에서 변경)
    - **FRONT VIEW**
    - **RIGHT / LEFT VIEW**
    - **TOP VIEW**
- 버튼 출력 순서를 상기 순서대로 조정합니다.

### 2.3. 실행 버튼 텍스트 변경
- 초기 상태(`!cvHasAnalyzed`): 실행 버튼 텍스트를 **'ANALYSIS'**로 변경합니다. (기존 'ANALYZE')
- 분석 중: 로딩 상태 표시 유지.
- 분석 완료 후: **'GENERATE'** 텍스트로 전환합니다.

## 3. 세부 구현 사항 (`App.tsx`)
- `cvHasAnalyzed` 상태에 관계없이 VIEWPOINT, ANALYSIS, VIEW 섹션을 렌더링하도록 조건부 로직 수정.
- 분석 전(`!cvHasAnalyzed`)에는 ANALYSIS 내 값들을 `none`으로 하드코딩 혹은 상태 처리.
- 뷰 버튼의 경우 분석 전에는 `disabled` 상태로 두어 분석이 선행되도록 가이드 (선택 사항).
