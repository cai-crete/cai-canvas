# 작업지시서 V314 (최초 아트보드 스케치 활성화)

**작성일**: 2026-04-01
**버전**: V314 (V313 보완본)
**상태**: 완료 (반영 완료)

---

## 1. 개요 및 목표
V313에서 도입된 "Smart Target" 로직(`src` 또는 `motherId` 보유 시에만 스케치 허용)이 아무런 이미지가 없는 최초의 빈 아트보드(`artboard`)까지 스케치를 차단하는 문제를 해결했습니다. 

## 2. 주요 분석 내용
*   **원인**: 최초 생성된 아트보드는 `src`가 `null`이고 `motherId`도 없기 때문에 V313의 스케치 허용 조건을 충족하지 못함.
*   **해결**: `item.type === 'artboard'`인 경우를 예외적으로 항상 스케치 가능 대상으로 포함함.

# 작업지시서 V315 (인플레이스 편집 및 연결선 통합 정교화)

**작성일**: 2026-04-01
**버전**: V315 (V310.2 피드백 반영본)
**상태**: 검토 대기

---

## 1. 개요 및 목표
`generated` 및 `sketch_generated` 객체에서 `+` 버튼 클릭 시 새로운 객체를 생성하는 것이 아니라, 해당 객체의 타입을 즉시 승격(`upload`)시켜 그 자리에서 스케치가 가능하도록 하는 "인플레이스(In-place) 편집" 로직을 완벽히 정착시킵니다. 또한 승격 후에도 기존의 가로(Horizontal) 연결선이 유지되도록 인터랙션을 보강합니다.

## 2. 주요 현황 분석
*   **인플레이스 로직 점검**: 현재 이미 `setCanvasItems((prev) => prev.map(...))` 방식을 사용하고 있으나, 특정 상황에서 중복 생성되거나 위치가 변할 가능성이 있는지 전수 조사 필요.
*   **연결선 방향 유실**: 현재 `generated`가 `upload`로 변하면 타입 기반의 `isHorizontalConnection` 판정 로직에서 제외되어 연결선이 세로로 꺾일 위험이 있음.

## 3. 주요 업데이트 계획

### ✅ 1. '+' 버튼 인플레이스 전환 로직 강화
| 구분 | 변화 전 (Derived) | 변화 후 (Editable) |
| :--- | :--- | :--- |
| **이미지 타입** | `sketch_generated` / `generated` | **`upload`** (승격) |
| **파라미터** | AI 생성 메타데이터 보유 | **`null`** (라벨 제거) |
| **계보(motherId)** | 모체 ID 보유 | **기존 ID 유지** |
| **동작** | 새로운 카드 생성하지 않음 | **그 자리에서 속성만 교체** |

### ✅ 2. 연결선(Bezier) 가로 방향 유지 (Fixed Horizontal)
*   **조건 보강**: 타입이 `upload`로 바뀌더라도 `motherId`를 보유한 "파생 업로드" 아이템인 경우, 계속해서 **가로(Horizontal)** 연결선을 유지하도록 `isHorizontalConnection` 로직을 수정합니다.
*   **출발/도착점 고정**: Mother 우측 중앙 $\rightarrow$ Child 좌측 중앙 설정을 강제합니다.

### ✅ 3. UI 자동화 (버튼 숨김)
*   `type: upload` + `motherId !== null`인 상태에서는 오버레이 툴바에서 `+` 버튼을 제거하여 중복 진입을 방지합니다.
