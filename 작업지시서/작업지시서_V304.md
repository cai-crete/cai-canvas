## 1. 파이프라인 분리 및 트리거 구조 개편

"Integration Process.md"의 런타임 오염 문제를 해결하기 위해, 하나의 버튼에 묶여있던 로직을 우측 제어 패널의 독립된 **2개의 탭(Tab)**으로 완벽히 분리합니다.

### A. 신규 탭: [IMAGE TO ELEVATION] (PHASE 2 전담)
*   **역할**: 1장의 이미지로부터 5면 정사영 전개도(ELEVATION SHEET)를 추출하는 기능 전담.
*   **UI 배치 및 스타일**: 우측 패널의 기존 "CHANGE VIEWPOINT" 탭(버튼)과 "PRINT" 버튼 사이에 신규 생성합니다. 버튼 스타일은 기존 패널 버튼들의 스타일링 시스템을 그대로 계승하여 이질감 없이 배치합니다.
*   **작동 방식**: 별도의 거창한 분석 패널(추가 패널)을 열지 않고, 클릭 트리거 시 백그라운드에서 바로 생성을 수행하여 기존 방식대로 깔끔하게 'ELEVATION SHEET'를 캔버스에 띄웁니다.
*   **독립성**: 이 탭의 연산은 시점 변경(투시)과 관련된 프롬프트가 전혀 섞이지 않으며, 오직 `1_Geometry`와 `2_Property` 마스터 파라미터를 추출하고 5면도를 생성하는 데에만 쓰입니다.

### B. 기존 탭: [CHANGE VIEWPOINT] (PHASE 3 전담)
*   **역할**: 사용자가 지정한 시점(투시 구도)으로 단 한 장의 이미지를 렌더링하는 시뮬레이션 전담.
*   **트리거 개편**: 탭 클릭 또는 진입 시, 더 이상 전체 파라미터를 처음부터 다시 훑는 '구 PHASE 1'을 발동하지 않고, 즉시 **PHASE 3 (시점 인지 및 구도 분석)** 로직만 발동합니다.
*   **파라미터 표시**: 업로드된 원본 이미지의 '현재 시점(V0)'을 분석하여 초기 상태 값으로 즉시 표시합니다.
*   **렌더링 실행**: 사용자가 앵글/고도를 변경한 후 하단의 [GENERATE]를 누르면 시점 렌더링이 독립적으로 돌아갑니다.

## 2. 신규 기능: 가시권 영역(Visible Area) 추출 및 락인(Lock-in) 방어

CHANGE VIEWPOINT 탭 내에 투시 오염 및 환각(Hallucination) 방지를 위한 절대적 제약 조건을 신설합니다.

*   **가시권 분석 (Phase 3 기반)**: V0(원본 시점) 분석 시, 원본 사진에 노출된 건물의 실제 '가시 각도 범위(Visible Angular Range)'를 추출하여 시스템 메모리에 저장합니다. (예: 정면~우측면 일부만 노출 = 04:00 ~ 08:00 범위)
*   **파라미터 잠금 (UI Lock)**: 슬라이더 컨트롤러 로직을 수정하여, **가시권 영역 범위를 벗어나는 절대 시점은 아예 드래그 및 선택이 불가**하도록 마지노선(Min/Max Clamp)을 설정합니다.
*   **시각적 다이얼 UI (2px Line)**: 우측 패널의 둥근 `VIEWPOINT` 뷰포트 다이어그램 겉면(원호) 위에, 추출된 가시권 영역 구간만큼만 **검은색 2px 실선(곡선)**을 렌더링하여 사용자에게 직관적으로 조작 가능 범위를 안내합니다.

## 3. 신규 기능: 파생 이미지 'edit(+)' 동작 변경 (모체 대체 로직)

파생된 이미지(시뮬레이션 결과물 등) 상단 제어바에 있는 `edit(+)` 버튼의 동작 방식을 개편합니다.
*   **기존 로직**: 클릭 시 모체 이미지(원본) 우측 옆 공간에 파생 이미지를 추가로 배치.
*   **변경 로직**: 클릭 시 모체 이미지 옆에 덧붙이지 않고, 캔버스 상에 존재하는 **해당 파생 이미지의 모체 이미지 노드 자체를 현재 파생 이미지로 완전히 대체(Replace)**하도록 변경합니다. (위치 및 노드 치환)

## 4. 문서 정비 (Integration Process.md)

`Integration Process.md` 문서에서 역할을 다했거나 모순을 일으키는 구 버전 명세(PHASE 1, PHASE 4)를 파기합니다.

## 4. 확정된 적용 기술 (Q1 제안 1 승인)

가시권 영역 검은선 2px UI를 렌더링하기 위해 **SVG Arc 렌더링 기술**을 적용합니다.

*   `App.tsx` 내의 앵글 다이얼(`AngleGauge` 역할) 쪽에 `<svg>` 태그를 덧대어, 판별된 가시권 각도(Visible Angular Range)만큼 정확하게 계산된 `stroke-width="2"`의 SVG 호(Arc)를 렌더링합니다.
*   이를 통해 각도의 시작점과 끝점에 정확히 닿는 오차 없는 수학적 시각 제어 시스템을 구현합니다.
