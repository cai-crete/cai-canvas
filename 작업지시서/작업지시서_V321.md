# 작업지시서 V321: 아트보드 (스케치 포함) 다운로드 기능 구현

본 작업지시서는 빈 아트보드 위에 스케치(SVG 경로)를 그린 후 다운로드 버튼 클릭 시 아무것도 다운로드되지 않는 문제를 해결하고, 캔버스의 모든 드로잉 데이터를 병합하여 저장하는 기능을 구현하는 것을 목표로 합니다.

## 1. 분석 및 문제 파악

### 1.1 빈 아트보드 다운로드 불가 문제
- **현상**: 새로 생성된 하얀색 아트보드(Image src가 `null`인 상태)에 펜으로 여러 스케치를 그린 뒤 상단의 `[다운로드]` 아이콘을 클릭해도 아무 반응이 없거나 실패함.
- **원인 분석**: V320에 추가된 다운로드 버튼은 `<a>` 태그의 `href={item.src}` 속성을 기반으로 작성됨. 빈 아트보드는 `item.src`가 존재하지 않으며, 스케치 데이터(`type: 'path'`)는 `canvasItems` 배열 상에서 모체(`motherId`)에 종속된 별개의 객체로 존재함.
- **해결 방향**: 단순히 이미지 소스를 링크하는 방식에서 벗어나, HTML5 Canvas API를 활용하여 빈 배경과 그 위에 그려진 모든 스케치 패스(Path) 점 데이터들을 하나의 캔버스에 병합(Render)한 후 다운로드하는 로직을 신규 구현해야 함.

## 2. 수행 계획

### 2.1 스케치 병합 다운로드 유틸리티 (`exportArtboardToImage`) 기능 신설
- `App.tsx` 내에 선택된 캔버스 아이템의 다운로드를 담당할 헬퍼 함수 구현:
  1. **가상 캔버스 생성**: 대상 아트보드(`width`, `height`)와 동일한 크기의 오프스크린 `<canvas>` 생성.
  2. **배경 렌더링**: `item.src`가 있으면 `drawImage`로 먼저 깔고, 빈 아트보드면 흰색(#FFFFFF)으로 채움.
  3. **스케치 데이터 필터링**: 전체 `canvasItems` 배열을 순회하며 `motherId`가 다운로드 대상 아이템의 `id`와 일치하는 `type: 'path'` 객체를 모두 수집함.
  4. **경로(Path) 드로잉**: 수집된 객체들의 `parameters.points` 좌표 배열을 순회하며 `ctx.lineTo()`를 통해 실제 선을 가상 캔버스에 그림. (선 굵기, 색상 반영)
  5. **이미지 추출**: `canvas.toDataURL('image/png')`를 추출하여 임시 앵커(`<a>`) 태그로 자동 다운로드 트리거.

### 2.2 아트보드 컨트롤 바 다운로드 버튼 교체
- 기존 V320에서 생성한 앵커 태그(`<a>`)를 버튼(`<button>`)으로 교체.
- 버튼 `onClick` 이벤트에 위에서 작성한 `exportArtboardToImage(item)` 함수를 연결함.
