# 작업지시서 V322: 로컬 이미지 업로드 시점 서버 주소 분리 및 환경 설정 가이드

본 문서는 Vercel 서버 분리, 브랜치 기반 편집 워크플로우, 그리고 로컬 프로젝트 폴더 명칭 변경에 대한 최종 가이드를 포함합니다.

---

## 🚩 [섹션 4] 로컬 프로젝트 폴더 명칭 변경 (change-viewpoint -> cai-canvas)

현재 사용 중인 로컬의 메인 작업 폴더 이름을 원하시는 이름으로 변경하는 안전한 정량적 방법입니다.

| 항목 | 기존 명칭 | 변경 명칭 | 변경 위치 |
| :--- | :--- | :--- | :--- |
| **Project Root Folder** | `change-viewpoint` | **`cai-canvas`** | Windows 탐색기(파일 관리자) |

### 1단계: 편집기(커서/VS Code) 및 터미널 종료
- 현재 이 프로젝트를 열고 있는 모든 에디터(Antigravity 실행 창 포함)와 실행 중인 터미널(`npm run dev` 등)을 종료합니다. 
- (실행 중인 상태에서 폴더명을 바꾸면 '액세스 거부'가 발생하거나 도구 경로가 손실될 수 있습니다.)

### 2단계: 폴더 명칭 직접 변경
- `d:\OneDrive\바탕 화면\CRETE\` 경로로 이동합니다.
- `change-viewpoint` 폴더를 클릭하고 **F2**를 눌러 **`cai-canvas`** 로 이름을 변경합니다.

### 3단계: 프로젝트 재로그인 및 에이전트 연결
- 바뀐 이름의 **`cai-canvas`** 폴더를 에디터(Cursor/Antigravity)로 다시 엽니다.
- 내부의 `.git` 설정은 폴더명과 상관없이 유효하므로 별도의 Git 재설정은 필요하지 않습니다.

### 4단계: 설정 파일 이름 동기화 (권장)
- 폴더명이 바뀌었으므로 프로젝트 내부 메타데이터도 맞춰주면 좋습니다.
- **`package.json`** 파일을 열어 상단의 `"name": "change-viewpoint"` 라고 되어 있는 부분을 `"name": "cai-canvas"` 로 수정합니다.

## [최종 요약 안내]
1. **서버 분리**: `cai-canvas` (최신), `cai-change-viewpoint` (구버전 아카이브) 셋업 완료.
2. **브랜치 워크플로우**: `main` 과 `archive-v1` 브랜치를 오가며 각기 다른 서버를 독립적으로 유지보수 가능.
3. **폴더 명칭**: 위의 4단계를 거쳐 `cai-canvas`라는 깔끔한 이름의 로컬 환경을 구축하십시오.
