# 작업지시서 V309 (UI 컨트롤 잠금, 스케치 사용성 개선, 가시성 및 라벨링 제어)

## 1. 개요 및 목표
기존 4가지 V309 계획(중복 생성 억제 토스트, 시트 버튼 잠금, 스케치 드래그 오류 수정, 줌아웃 라벨 숨김)에 이어, 서로 다른 모체(Mother Image) 간의 독립적 라벨 넘버링(`VIEW 01, 02...`) 구조를 정립합니다.

## 2. 작업 상세 내용 (수행 계획)

### 🚩 1. "IMAGE TO ELEVATION" 중복 방지 토스트 알림
*   우측 툴바의 "IMAGE TO ELEVATION" 클릭 시, 선택된 이미지(`sourceItem`)를 기준으로 전개도가 이미 생성되었는지 검사합니다.
*   존재할 경우 실행을 중단(`return`)하고, 기존 토스트 창에 `'이미 전개도 생성이 완료된 이미지입니다.'` 메시지를 띄웁니다.

### 🚩 2. 'ELEVATION SHEET' 툴바 강제 잠금 (Disabled)
*   선택된 아이템의 라벨이 `ELEVATION SHEET`일 경우, 해당 시트(Sheet)를 대상으로는 동작할 수 없는 우측 메뉴인 "IMAGE TO ELEVATION", "CHANGE VIEWPOINT" 버튼 노드를 완전히 잠금(`disabled`) 상태로 렌더링합니다.

### 🚩 3. + 버튼 파생 편집 탭 강제 스케치 모드 및 "드래그 앤 드롭" 삭제
*   **이동 속성 박탈**: `handlePointerDown` 로직에서 파생된 이미지(`type: 'upload'` 이면서 `motherId` 존재)는 마우스 드래그에 의한 객체 이동(Drag & Drop) 추적 조건에서 완전히 분리 배제합니다.
*   **스케치 모드 즉각 진입**: 파생판의 `+` 버튼을 누르면 상태 변경과 동시에 글로벌 `setCanvasMode('pen')` 함수를 호출하여 캔버스가 즉시 펜 모드로 진입하도록 구현합니다.

### 🚩 4. ELEVATION SHEET 방향 라벨 줌-아웃 숨김 처리
*   ELEVATION SHEET에서 렌더링하는 `REAR`, `LEFT`, `TOP`, `RIGHT`, `FRONT` 라벨 렌더링 블록을 찾아 조건부 렌더링을 적용합니다.
*   `{canvasZoom > 40 && ...}` 형태로 조건을 감싸, 줌 배율이 40% 이하일 때는 리액트 렌더 트리에서 아예 제거되도록(Hidden) 깔끔하게 구현합니다.

### 🚩 5. 모체 독립적 'VIEW XX' 파생 넘버링 카운트 (추가됨)
*   **현상**: 'CHANGE VIEWPOINT' 파생뷰 생성 시 캔버스 전역을 기준으로 `VIEW ` 라벨 개수를 카운트하여 다른 객체의 자식 숫자까지 합산하는 오류가 존재.
*   **해결**: `App.tsx`의 뷰포인트 넘버링 로직(`vpCount`) 도출 시, 단순히 라벨 이름만 검사하는 것이 아니라 `ci.motherId === sourceItem.id` 조건을 필터링에 추가합니다. 이를 통해 각 모체별로 독립된 스택 공간을 인정하고, 새로운 모체에서 생성 시 항상 `VIEW 01`부터 시그니처가 시작되도록 제어합니다.
