# 작업지시서 V335: CHANGE VIEWPOINT 백엔드 연동 및 프로세스 분리

## 1. 개요
* **목표**: "CHANGE VIEWPOINT" 파이프라인의 구동 방식을 전면 개편하여 전용 백엔드 서버(`cai-change-viewpoint-v2`)와 연동합니다.
* **관련 앱**: PLANNERS, SKETCH TO IMAGE, PRINT와 동일한 방식의 백엔드 연동 아키텍처로 통일합니다.

## 2. 상세 작업 항목

### 2-1. `cai-change-viewpoint-v2` 백엔드 호스팅 연결 기반 수립
* **연결 엔드포인트**: `https://cai-change-viewpoint-v2.vercel.app/` 서버를 백엔드로 사용.
* **의존성 제거**: 
  - 기존 로직에서 "CHANGE VIEWPOINT"와 "IMAGE TO ELEVATION" 간의 결합 로직을 완전히 제거.
  - "IMAGE TO ELEVATION" 기능은 독립된 단일 프로세스 아키텍처로 완전히 분리.

### 2-2. UI 패널 트리거 조건 변경
* **현행 문제**: "IMAGE TO ELEVATION"이 생성(선택)되어야만 "CHANGE VIEWPOINT" 패널이 열림.
* **변경 사항**: 캔버스 상에서 **이미지만 선택해도** 바로 "CHANGE VIEWPOINT" 패널이 열리도록 설정 변경 및 가드 조건 해제.

### 2-3. UI 및 프로세스 플로우 개편 (ANALYZE ➜ GENERATE)
* **초기 버튼 렌더링 변경**: 
  - 입력창의 최초 버튼을 `GENERATE`에서 `ANALYZE`로 변경하여 출력합니다.
* **ANALYZE 프로세스**: 
  - `ANALYZE` 버튼 클릭 시, 프론트엔드(`cai-canvas`)에서 백엔드(`https://cai-change-viewpoint-v2.vercel.app/`)로 분석 요청을 전송.
  - 리턴 결과값은 `cai-change-viewpoint-v2` 프론트엔드 앱에서 'ANALYZE' 버튼을 클릭했을 때와 완전히 일치하는 구조로 설정.
* **분석 결과 저장 및 동기화**: 
  - 반환된 분석 결과를 `cai-canvas` 스토어와 백엔드 양측 모두에 동기화 및 저장합니다.
* **최종 이미지 생성 절차 (GENERATE)**: 
  - 분석이 끝난 후 사용자가 추가적으로 선택 및 입력한 정보(뷰포인트, 위치 값, 프롬프트 등)를 모아 `cai-change-viewpoint-v2` 백엔드로 송신하여 이미지를 최종 생성.
  - 결과물을 응답받아 `cai-canvas`에 노드(Node) 형태로 최종 출력.

## 3. 작업 진행 상태 
* **승인 대기 중**: 상기 내용을 바탕으로 사용자 작업 승인 후 `App.tsx` 등 관련 컴포넌트의 구현(Phase 4)을 시작합니다.
