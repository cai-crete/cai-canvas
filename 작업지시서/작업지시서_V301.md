# 작업지시서 V301 (입면도 뷰 파생 알고리즘 개선 및 TOP 뷰 정상화)

**작성일**: 2026-03-31
**버전**: V301
**상태**: 승인 대기 (Q1 확인 중)

---

## 1. 공간 지각(Spatial Awareness) 파생 로직 분석

요청하신 "원본 이미지와 출력한 전개도 사이의 관계 및 각 뷰를 파악하는 방식"은 다음과 같은 3단계 원리로 작동합니다.

### 1단계: 정면(FRONT, 06:00)의 논리적 인식 (Phase 0)
AI는 원본(Source) 이미지의 픽셀을 단순히 2D로 복사하는 것이 아니라, 건축적 맥락(주출입구의 위치, 메인 파사드의 비례, 창문의 정면성 등)을 분석하여 **"건물의 진짜 정면이 어느 쪽인가?"**를 추론합니다. 

### 2단계: 기준점(Anchor) 설정 (Step 1)
추론된 진짜 정면을 공간 상의 **[방위각(Azimuth) 0°, 고도(Altitude) 0°]** 지점으로 고정(Lock)하고, 이를 V300에서 추가했던 `FRONT 앵커 이미지`로 시각화하여 기준을 세웁니다.

### 3단계: 상대적 회전에 따른 뷰 파생 (Step 2)
정면(FRONT)이 확립되면, 이 기준점을 중심으로 카메라를 회전시켜 나머지 뷰를 파생합니다.
- **RIGHT (우측면)**: 정면에서 우측으로 90° 이동 (`Azimuth 90°`)
- **REAR (배면)**: 정면에서 180° 뒤로 이동 (`Azimuth 180°`)
- **LEFT (좌측면)**: 정면에서 좌측으로 90° 이동 (`Azimuth 270°`)
- **TOP (지붕/평면)**: 정면 위치에서 수직 위로 90° 상승하여 내려다봄 (`Altitude 90°`)

---

## 2. 현 상황의 문제점 추적 (TOP 뷰 실패 원인)

**현상**: 스크린샷 확인 결과, TOP 뷰 자리에 지붕 평면도가 아닌 정면/투시도와 유사한 이미지가 출력되고 있습니다. LEFT/RIGHT/REAR도 각도가 완벽한 정투상(Orthographic) 입면도라기보다는 살짝 투시가 섞이거나 정면의 잔재가 남아 있습니다.

**원인 (V300의 스택링 부작용)**:
V300에서 형태 유지를 위해 모든 파생 뷰에 **"FRONT 앵커(정면 이미지)"**를 강제로 쥐여주었습니다. 
정면 입면도는 구조상 "지붕(Roof)" 데이터가 선(Line) 하나로 압축되어 있어 보이지 않습니다. 
AI에게 "이 정면 이미지를 똑같이 참고해서 TOP 뷰(수직으로 내려다본 뷰)를 그려라"라고 지시하면, 물리적 모순이 발생하여 AI가 고도 상승(Altitude 90) 명령을 무시하고 그냥 눈앞에 보이는 정면/투시 뷰를 다시 그려버리게 됩니다.

## 3. 해결을 위한 제안 (Q1)

우선 TOP 뷰는 구조가 본질적으로 다르므로 전략을 분리해야 합니다.

### 제안 1 (뷰 특성에 따른 앵커 분리 적용 - 권장)
- **TOP 뷰 생성 시**: FRONT 앵커 이미지를 **제거**합니다. TOP 뷰는 오직 '원본 이미지(지붕의 원근감이 포함된 사진)'만을 참조하여 형태적 모순 없이 지붕 뷰를 만들도록 합니다.
- **RIGHT / LEFT / REAR 뷰 생성 시**: FRONT 앵커 이미지를 **유지**하되, 프롬프트의 강제성을 수정합니다. "FRONT 이미지와 똑같은 건물을 그리되, **절대로 정면을 그리지 말고 측면/후면만 그려라**"라고 명시하여 정면의 시각적 잔재를 지웁니다.

### 제안 2 (모든 뷰에서 FRONT 앵커 제거 후 프롬프트 기반 텍스처/형태 분리)
모든 뷰에서 V300의 2-Step FRONT 앵커 방식을 포기하고, 다시 원본 이미지 1장만 넘기되 프롬프트를 혁신적으로 변경합니다. "이미지는 재질용이고, 형태는 완전히 부수고 주어진 각도로 새로 모델링하라"는 지시를 넣습니다. (단, V299처럼 다른 건물이 나올 위험이 큼)

### 제안 3 (FRONT 앵커의 투명도/가중치 논리적 무효화)
API 스펙상 이미지 투명도를 조절할 순 없으므로, 프롬프트 상단에 "IMAGE 2(FRONT 앵커)는 '건물의 층고와 좌우 비례'를 맞추는 실루엣 용도로만 쓰고, 외관 형태는 무시하라"고 제한적인 역할만 부여합니다.
