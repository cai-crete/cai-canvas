# 작업지시서 V313 (스케치 인터렉션 최종 보정 및 드래그 완벽 차단)

**작성일**: 2026-04-01
**버전**: V313 (V310 피드백 반영본)
**상태**: 검토 대기

---

## 1. 개요 및 목표
`+` 버튼 클릭 후 파생 이미지에서 스케치가 원활하게 되지 않고, 여전히 브라우저 기본 드래그 속성이 남아 인터렉션을 방해하는 문제를 해결합니다. "드래그 앤 드롭 관련 모든 로직 삭제" 요청에 따라 객체의 이동 특성을 제거하고 순수하게 "스케치 보드"로서 작동하도록 고정하는 것을 목표로 합니다.

## 2. 주요 분석 내용
*   **드래그 속성 간섭**: `type: 'upload'` 상태에서 브라우저 기본 `draggable={true}` 속성이 활성화되어 있어, 마우스 클릭 시 그림을 그리는 대신 이미지를 끌어오려는(Drag) 동작이 우선 발생함.
*   **그리기 대상 누락**: `handlePointerDown`의 `pen` 모드 로직에서 `sketch_generated` 및 특정 상황의 `artboard`가 스케치 허용 대상에서 누락되어 있음.
*   **상태 전이 불일치**: `+` 버튼 클릭 시 타입은 변경되지만, HTML 속성 레벨에서의 드래그 옵션이 해제되지 않아 충돌 발생.

## 3. 주요 업데이트 계획

### ✅ 2. 정밀한 스케치 영역 활성화 (Smart Target)
*   **유효 영역 판정**: `pen` 및 `eraser` 모드에서 클릭 시, 단순히 타입을 체크하는 것이 아니라 `src`가 존재하거나 `motherId`가 있는('유효 편집 영역') 객체가 있는지를 확인합니다.
*   **자동 Pan 전환**: 유효한 객체 위가 아닌 빈 공간(배경 도화지 포함)을 클릭할 경우 즉시 화면 이동(`pan`) 모드로 작동하여 원치 않는 낙서를 방지합니다.

### ✅ 3. '+' 버튼 클릭 시 상태 변화 상세 분석
파생 이미지(`sketch_generated`, `generated`) 내의 `+` 버튼 클릭 시 발생하는 변화는 다음과 같습니다.

| 구분 | 변화 전 (Derived) | 변화 후 (Editable) |
| :--- | :--- | :--- |
| **이미지 타입** | `sketch_generated` 또는 `generated` | **`upload`** (수정 가능 타입으로 승격) |
| **파라미터** | AI 생성 옵션 및 메타데이터 보유 | **`null`** (내비게이션 `< >` 라벨 제거 목적) |
| **커서/모드** | `select` 또는 기존 모드 유지 | **`pen` (스케치 모드 즉시 전환)** |
| **계보(motherId)** | 모체 아이디 보유 | **기존 motherId 유지** (연결선 보존) |
| **오버레이 UI** | `+`(편집), `Download`, `Delete` 노출 | **`+` 버튼 자동 제거**, 나머지 기능 유지 |

*   **UI 자동화**: `type`이 `upload`이면서 `motherId`가 있는 경우, 이미 편집 모드에 진입한 것으로 간주하여 오버레이 툴바에서 `+` 버튼이 더 이상 나타나지 않게 제어됩니다. (중복 생성 방지)
*   **연결선(Bezier)**: `motherId`가 유지되므로 캔버스 상의 시각적 연결 고리는 끊어지지 않고 그대로 유지됩니다.

### ✅ 5. 연결선(Bezier) 방향 및 위치 자동 최적화
객체 간의 관계를 명확히 시각화하기 위해 타입/라벨별 연결 방향을 강제합니다.

| 대상 객체 | 연결 방향 | 출발 위치 (Mother) | 도착 위치 (Child) |
| :--- | :--- | :--- | :--- |
| **sketch_generated** | **가로 (Horizontal)** | 우측 중앙 (Right Center) | 좌측 중앙 (Left Center) |
| **ELEVATION SHEET** | **세로 (Vertical)** | 하단 중앙 (Bottom Center) | 상단 중앙 (Top Center) |

*   **배경**: 기존에 `artboard`나 `generated` 타입에 대해 일률적으로 적용되던 가로 연결 로직을 세분화하여, 도면 시트(`ELEVATION SHEET`)는 직관적인 상하 구조(Vertical)를 갖도록 하고, 스케치 결과물은 좌우 병렬 구조(Horizontal)를 유지하도록 보강합니다.

## 4. 현 기능 대비 변경 사항 상세 (Option 1 적용 시)

| 항목 | 현재 상황 (V312 기준) | V313 적용 후 변경 사항 |
| :--- | :--- | :--- |
| **드래그 방식** | 브라우저 기본 드래그(`draggable=true`) 활성화로 인한 '이미지 끌어옴(Ghosting)' 현상 발생. | 브라우저 기본 드래그를 삭제하고 **순수 JS 방식**의 부드러운 커스텀 드래그만 남음. |
| **이동 범위** | V309에서 차단되었던 '파생 이미지 이동 불가' 로직이 혼재되어 이동이 매끄럽지 않음. | `select` 모드에서 원본/파생 구분 없이 **모든 객체의 자유로운 이동** 보장. |
| **스케치 대상** | `type` 기반 체크(`upload`, `generated`만 허용)로 인해 `sketch_generated` 등에서 누락 발생. | `src` 유무 및 `motherId` 유무를 기준으로 **유효한 모든 이미지 위에서 스케치** 가능. |
| **빈 공간 클릭** | 빈 아트보드나 배경 클릭 시 에러 혹은 무반응(또는 강제 모드 전환) 가능성. | 유효 영역 외 클릭 시 **즉시 화면 이동(`pan`)으로 자동 전환**되어 드로잉 실수 방지. |

## 5. 수행 계획 및 확인 사항
1.  **Item Wrapper 수정**: `draggable={false}` 설정 및 `onDragStart` 이벤트 핸들러 제거.
2.  **handlePointerDown 보강**:
    *   `select` 모드: 모든 이미지(`type !== 'path'`) 드래그 허용 로직 정리.
    *   `pen/eraser` 모드: `drawTarget` 탐색 조건에 `item.src || item.motherId` 적용.
3.  **빌드 검사**: `tsc --noEmit`을 통한 정적 타입 체크.
4.  **최종 검증**: 펜 모드에서 이미지 위 스케치 vs 빈 공간 Pan 작동 여부 교차 테스트.
