# 작업지시서 V320: UI 개선 및 스케치/드래그 로직 최적화

본 작업지시서는 우측 패널의 시각적 피드백 개선, 아트보드 컨트롤 기능 확장, 그리고 파생 이미지에서 발생하는 스케치 동작 오류 수정을 목표로 합니다.

## 1. 분석 및 문제 파악

### 1.1 우측 패널 버튼 호버 효과 (UI/UX)
- **현상**: WRITER, STYLE EDIT 등 현재 기능이 할당되지 않은 플레이스홀더 버튼들에 마우스 오버(Hover) 시 시각적 변화가 없어 사용자에게 버튼의 존재감이 약함.
- **해결**: 플레이스홀더 버튼도 일반 버튼과 동일하게 배경색이 미세하게 변하는 효과를 적용하여 일관된 UI 경험 제공.

### 1.2 아트보드 컨트롤 바 기능 추가
- **현상**: 초기 생성된 아트보드에는 `[upload]ㅣ[lock]ㅣ[delete]` 기능만 존재함.
- **해결**: 아트보드 작업 결과를 로컬에 저장할 수 있도록 **[download]** 버튼을 중앙에 추가하여 `[upload]ㅣ[lock]ㅣ[download]ㅣ[delete]` 순서로 확장.


## 2. 수행 계획

### 2.1 우측 패널 호버 효과 적용
- `App.tsx` 내 버튼 렌더링 루프(약 4033행)에서 `isPlaceholder` 조건에 상관없이 `hover:bg-black/5` 스타일이 적용되도록 CSS 클래스 수정.

### 2.2 아트보드 [download] 버튼 추가
- 아트보드 플로팅 툴바(약 3520행 부근)에 `Download` 아이콘 버튼 및 구분선(`Separator`) 삽입.
- `<a>` 태그의 `download` 속성을 활용하거나 `handleDownload` 유틸리티 연동.

## 3. 검증 계획
- [ ] 우측 패널의 FUNCTION 버튼들에 마우스를 올렸을 때 배경색이 변하는지 확인.
- [ ] 아트보드 상단 바에 다운로드 아이콘이 생성되었고 클릭 시 이미지가 저장되는지 확인.
