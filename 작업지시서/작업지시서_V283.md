### Change A: z-index — selection border 섹션 레벨 격상

#### 원인 분석

```
<section> (캔버스 영역)
  <div 통합 오버레이 z-[100]>  ← 스케치/Lasso/연결선
  <div 캔버스 레벨 로더 z-[102]>  ← V282 (isAnalyzing||isGenerating)
  <div transform-wrapper (transform 有, z-index 無 = z-auto)>
    ← z-auto 전체는 섹션 stacking context에서 z-102 로더보다 아래
    <div canvas-item z-20> (selected)
      <div selection-border z-[105]>  ← 로컬 z만, 로더에 가려짐 ✗
      <div 제어바 z-[40]>             ← 로컬 z만, 로더에 가려짐 ✗
```

#### 해결 방안

selection border(파란 테두리)를 섹션 레벨 별도 오버레이로 중복 렌더링 (z-[105]).
기존 아이템 내부 border는 유지 (비로딩 상태에서 정상 동작).

**A-1: 선택 border 오버레이 삽입** — 로더 오버레이 바로 뒤 (~line 2657):

```tsx
{/* V283 A: 섹션 레벨 selection border 오버레이 (z-105) — 로더(z-102) 위에 표시 */}
{selectedItemIds.length > 0 && (
  <div
    className="absolute inset-0 pointer-events-none z-[105]"
    style={{
      transform: `translate(calc(50% + ${canvasOffset.x}px), calc(50% + ${canvasOffset.y}px)) scale(${canvasZoom / 100})`,
      transformOrigin: '0 0'
    }}
  >
    {selectedItemIds.map((itemId: string) => {
      const selItem = canvasItems.find((i: CanvasItem) => i.id === itemId);
      if (!selItem || selItem.type === 'path' || selItem.type === 'text') return null;
      return (
        <div
          key={`sel-border-${itemId}`}
          style={{
            position: 'absolute',
            left: selItem.x - 1,
            top: selItem.y - 1,
            width: selItem.width + 2,
            height: selItem.height + 2,
            borderWidth: `${1.6 / (canvasZoom / 100)}px`,
            borderStyle: 'solid',
            borderColor: '#1d4ed8',
          }}
        />
      );
    })}
  </div>
)}
```

> 기존 아이템 내부 selection border(`~line 2963 z-[105]`)는 **그대로 유지**.

### Change B: EDITED 01 연결선 — motherId 수정

#### 원인

`~line 2005`:
```ts
const targetMotherId = sourceItem ? (sourceItem.motherId || sourceItem.id) : null;
```

- EDIT 01의 `motherId` = GENERATED(sketch_generated)의 id
- 위 코드: `sourceItem.motherId` 우선 → EDITED 01의 motherId = GENERATED id
- 결과 연결선: GENERATED → EDITED 01 ✗

#### 수정

```ts
// V283 B: EDITED 01 연결선이 EDIT 01에 연결되도록 sourceItem.id 직접 사용
const targetMotherId = sourceItem?.id ?? null;
```

- EDITED 01의 `motherId` = EDIT 01 id → 연결선: EDIT 01 → EDITED 01 ✓

### Change D: VIEWPOINT 01 연결선 — motherId 직접 수정

#### 원인

`~line 1807`:
```ts
motherId: sourceItem.motherId || sourceItem.id,
```

- sourceItem이 모체(motherId) 없는 upload → `motherId = sourceItem.id` ✓
- sourceItem이 모체 있는 artboard → `motherId = sourceItem.motherId` → 조부모를 가리킴 ✗

#### 수정

```ts
// V283 D: VIEWPOINT 01 연결선이 ANALYZED 모체에 직접 연결되도록
motherId: sourceItem.id,
```

`~line 1822` siblings 필터도 단순화:
```ts
// 변경 전
const siblings = prev.filter(i => i.type === 'generated' && (i.motherId === sourceItem.id || i.motherId === sourceItem.motherId));
// 변경 후
const siblings = prev.filter((i: CanvasItem) => i.type === 'generated' && i.motherId === sourceItem.id);
```
